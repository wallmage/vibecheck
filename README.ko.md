매일 오후만 되면 Claude 할당량이 바닥났는데 이유를 알 수 없었습니다. 알고 보니 AI 코딩 세션의 70%가 낭비였습니다 — AI가 지금부터 할 일을 설명하고, 한 번에 끝낼 수 있는 명령어를 세 턴에 걸쳐 나누고, 오래된 컨텍스트가 쌓여서 매 턴마다 다시 읽히고 있었습니다.

vibecheck은 그 낭비를 찾아냅니다. 24개 이상의 코딩 도구에서 실제 세션 로그를 읽고, 15가지 구체적인 패턴에 금액을 매기고, 수정합니다. 모든 것이 로컬에서 실행됩니다. 업로드 없음, 텔레메트리 없음, 서버 없음.

제 경우: 월 비용이 $2,816에서 $422로 줄었습니다. **85% 절감.**

## 설치 방법

AI 코딩 도구에 이것을 붙여넣고 Enter를 누르세요:

> Help me install this skill: https://github.com/wallmage/vibecheck

끝입니다. AI가 skill을 로드하면 바로 스캔할 수 있습니다.

### "skill"이 정확히 뭔가요?

AI에게 새로운 것을 가르치는 순수 텍스트 파일입니다. 바이너리 없음, 백그라운드 프로세스 없음, 시스템 수정 없음. vibecheck의 skill 파일은 "낭비를 찾고 수정하는 방법"을 설명합니다. 폴더를 삭제하면 사라집니다.

### 코딩 도구 vs. 채팅 도구

**코딩 도구** (Claude Code, Codex, Cursor, OpenClaw, Copilot, Windsurf, TRAE, Qoder 등)는 로컬에서 실행되며 세션 로그를 남깁니다. vibecheck은 설치된 도구를 자동으로 감지하고 해당 로그를 직접 스캔합니다.

**채팅 도구** (Cowork, 브라우저 기반 Claude)는 로컬 로그 없이 샌드박스에서 실행됩니다. vibecheck은 여전히 인스트럭션 파일을 최적화합니다 — 실제로 절감의 대부분은 거기서 나옵니다. 터미널 명령어 하나를 붙여넣으면 14일치 로그를 내보내 전체 스캔도 가능합니다.

### 권한

vibecheck은 로컬 세션 로그를 읽고 인스트럭션 파일(`CLAUDE.md`, `AGENTS.md`, `Memory.md`)과 시스템 전체 도구 설정을 검사합니다. 도구에 글로벌 설정 — 모든 프로젝트에 적용되는 하나의 파일 — 이 있으면 옵티마이저가 그것부터 처리합니다. 한 곳을 수정하면 모든 곳에서 비용이 절감되기 때문입니다. 무엇이든 변경하기 전에 먼저 확인을 요청합니다.

## 프라이버시

모든 것이 로컬에 남습니다. 분석은 로컬 세션 로그를 파싱하는 Python 스크립트입니다. 서버 없음, API 호출 없음, 분석 수집 없음. 오픈 소스 — 원하면 모든 줄을 확인하세요.

## 명령어

| 명령어 | 기능 |
|---|---|
| `/vibecheck scan` | 머신에서 감지된 모든 도구를 스캔. 건강 지표, 순위 통계, 주요 낭비 패턴, 최적화 로드맵을 포함한 통합 보고서 |
| `/vibecheck explain` | 아무것도 변경하지 않고 낭비 패턴을 설명. 순수 학습용 |
| `/vibecheck compress` | 4단계 무손실 압축기로 인스트럭션 파일을 25-50% 축소 |
| `/vibecheck monitor` | 베이스라인과 주간 비교. 비용 회귀를 누적되기 전에 감지 |

스캔은 조용하게 진행됩니다: 간결한 진행 표시기 다음에 깔끔한 요약. `Good`은 측정된 낭비가 10% 미만, `Waste`는 그 이상을 의미합니다. 원시 로그와 도구 출력은 문제가 없는 한 표시되지 않습니다.

### 세션을 신선하게 유지하기

긴 대화는 짧은 대화보다 비쌉니다 — 새 턴마다 이전의 모든 턴을 다시 읽고, 과부하된 컨텍스트는 AI를 부정확하게 만들어 더 많은 왕복이 필요해집니다.

경험 법칙: 세션당 5-10분의 실작업, 30-40턴 이후부터 컨텍스트 세금이 본격적으로 느껴집니다. 새 세션을 시작할 때는 지속적인 규칙을 인스트럭션 파일(`CLAUDE.md`, `AGENTS.md`, `Memory.md`)에, 프로젝트 배경을 작은 로컬 문서에 넣으세요. 새 세션은 처음부터 다시 시작하는 것이 아닙니다 — 모든 지식이 그대로 있는 깨끗한 컨텍스트일 뿐입니다.

## AI 턴이 비용을 발생시키는 원리

토큰 경제학을 한 번도 생각해본 적 없는 분을 위한 간단한 입문. AI 가격 배경지식은 필요 없습니다.

### 매 턴마다 무슨 일이 일어나는가

AI가 응답할 때마다 대화 전체를 처음부터 다시 읽습니다. 시스템 프롬프트, 인스트럭션 파일, 여러분이 보낸 모든 메시지, AI의 모든 응답, 모든 도구 출력 — 파일 내용, 터미널 결과, 에러 로그 — 전부. 그런 다음 새 응답을 생성합니다.

**턴 비용 = 읽은 token 수 x 입력 가격 + 생성한 token 수 x 출력 가격**

초기 턴은 저렴합니다. 턴 1은 5,000 token 정도를 읽을 수 있습니다. 턴 40에서는 축적된 대화의 40,000개 이상의 token을 다시 읽습니다 — 이전의 모든 메시지, 코드 스니펫, 에러 트레이스. 그 후반 턴은 첫 번째 턴의 8배 비용입니다.

핵심은 낭비가 **복리로 누적**된다는 것입니다. 낭비된 턴은 자체 token 비용만 발생하는 것이 아닙니다. 세션의 나머지 동안 컨텍스트에 남아서 이후의 모든 턴에서 다시 읽힙니다. 턴 10에서의 불필요한 설명 메시지 하나가 세션이 끝날 때까지 30번 더 다시 읽힙니다.

### 프롬프트 캐시는 도움이 되지만 근본적 해결은 아닙니다

대부분의 제공업체는 이제 이전에 본 콘텐츠를 캐시하고 10배 적게 청구합니다. 유효 입력 비용이 $3.00/백만 token에서 $0.30/백만으로 떨어집니다.

도움이 됩니다. 하지만 새 콘텐츠 — 새 도구 출력, 새 에러 메시지, 매번 새로운 AI 응답 — 는 캐시되기 전에 항상 정가로 들어옵니다. 그리고 캐시 요금에서도 낭비는 여전히 복리로 누적됩니다.

### 구독도 같은 고통을 겪습니다

구독 중이라면 API 가격이 자신과 무관하다고 생각할 수 있습니다. 관련이 있습니다 — 느끼는 방식이 다를 뿐입니다. 구독은 고정된 컴퓨트 풀을 구매하는 것이고, 낭비는 그 풀을 더 빨리 소모합니다. 오후 3시에 할당량에 도달해 속도 제한을 받는 것은 너무 많이 일해서가 아닙니다 — 그 일의 상당 부분이 낭비였기 때문입니다.

Claude Pro ($20/월)는 API 환산 약 $200에 해당합니다. Claude 20x Max ($200/월)는 약 $4,000에 해당합니다. 낭비가 많을수록 = 벽에 더 빨리 부딪힙니다.

### 실제 사용감

- **$20 Claude Pro** — 본격적인 개발 작업(기능 구현, 리서치, 문서 작성)은 5시간 할당량을 1시간 이내에 소진. 주간 용량 7시간 미만. 전문 용도로는 빠듯합니다.
- **$100 5x Max** — 약 4시간이면 5시간 윈도우에 도달. 주간 합계 30-35시간. 일반 사용에 적합합니다.
- **$200 20x Max** — 주 80-100시간 이상 일하며 여러 세션을 병렬로 실행하는 사람을 위한 플랜.

### Anthropic이 서드파티 구독 사용을 제한한 이유

10-20배 액면가에서 구독 1달러당 API 요금보다 훨씬 많은 컴퓨트를 구매하는 셈입니다. 서드파티 도구가 API 환산 속도로 소모하니 수지가 맞지 않게 되었습니다.

### Codex 대안

$20 플랜에서 Codex Plus는 Claude Pro의 약 **3배 코딩 사용량**을 제공합니다. ChatGPT 대화 — GPT-5.4 Extended Thinking과 딥 리서치까지 포함 — 는 Codex 코딩 할당량에 포함되지 않습니다. 즉 3배의 코딩 용량에 무료 GPT-5.4가 추가됩니다.

**$20/월 예산이라면 Codex Plus가 Claude Pro보다 코딩 시간이 더 많습니다.** 더 많이 쓸 수 있다면 Claude 5x와 20x가 다른 가치를 제공합니다.

## 15가지 낭비 패턴

비용 영향이 큰 순서로 정리했습니다. 상위 3개만으로 전체 낭비의 60-70%를 차지합니다.

### Tier 1 — 3대 패턴 (낭비의 60-70%)

#### 1. 유휴 내레이션

AI가 "네, 지금 수정하겠습니다" 또는 "먼저 파일을 읽어보겠습니다"라고 말하고 — 실제 작업은 다음 턴에서 합니다. 그 내레이션 턴은 아무것도 하지 않았습니다. 도구 호출 없음, 코드 없음, 파일 읽기 없음. 그냥 예고.

각각 약 **$0.017** — 더 나쁜 것은 그 300-500 token의 상태 텍스트가 컨텍스트에 남아 이후 모든 턴에서 다시 읽힌다는 것입니다. 428개 세션 측정 결과: **세션당 $1.03 낭비**, 전체 낭비의 30%. 하루 10세션이면 **내레이션만으로 월 $309.**

vibecheck 규칙: *"도구 호출 없는 턴은 금지. 같은 턴에서 생각하고 행동하라."* **세션당 약 $0.88 절약.**

#### 2. 컨텍스트 부패

세션 비용은 선형이 아니라 이차함수적으로 증가합니다. 턴 50은 이전 49개 턴 전부를 다시 읽습니다.

구체적 비교: 40턴 세션 1회는 **$0.70**. 같은 작업을 20턴 세션 2회로 나누면 **$0.60**. 그 $0.10 차이는 부풀어진 대화를 유지하는 순수한 낭비. 실제 세션은 평균 74턴 — **세션당 $0.46 낭비**, 전체 낭비의 13%.

vibecheck이 가르치는 것: *"관련 없는 작업은 별도 세션으로. 긴 스레드에서는 간결하게."* **세션당 약 $0.37 절약.**

#### 3. 핑퐁 디버깅

수정, 깨짐, 재시도, 또 깨짐. 매 실패 시도마다 약 4,000 token의 에러 출력이 컨텍스트에 쌓이고, 그 쓸모없는 텍스트가 이후 모든 턴에서 다시 읽힙니다. 3회 사이클: 6개 추가 턴 ($0.102) + 12K token의 오래된 에러 ($0.036) = **에피소드당 약 $0.14**. 세션의 약 10%에서 발생. **가중치: 세션당 $0.015.**

vibecheck 서킷 브레이커: *"같은 파일에서 수정 2회 실패 후 — 멈추고, 전체 에러를 다시 읽고, 생각하고, 정확한 수정 1회."* **세션당 약 $0.01 절약.**

### Tier 2 — 턴 밀도 (낭비의 15-20%)

한 턴에 할 수 있는 일을 세 턴에 걸쳐 하기.

#### 4. 장황한 도구 출력

빌드나 테스트 명령이 500줄(약 5,000 token)을 대화에 쏟아냅니다. 그 token은 세션의 나머지 모든 턴에서 다시 읽힙니다. 5K token x 12개 남은 턴 x 캐시 요금 = **인스턴스당 $0.018**. 캐시 없으면: **$0.180** — 10배.

측정상 가장 비용이 높은 단일 패턴입니다. 빌드 로그, npm 출력, 테스트 덤프 — 거의 모든 세션에 영향을 미칩니다. **세션당 $1.05**, 전체 낭비의 31%.

수정: *"출력을 /tmp/로 파이프. --quiet 플래그 사용. tail -50 최대."* **세션당 약 $0.89 절약.**

#### 5. 체인되지 않은 명령어

`npm install`을 한 턴. `npm run build`를 다음 턴. `npm install && npm run build`면 한 번에 끝나는 것을 두 번 컨텍스트 재읽기. 분할당: **$0.010**. 명령어 집약 세션에서 합계 **세션당 $0.14**.

수정: *"안전할 때 `&&`로 체인."* **세션당 약 $0.11 절약.**

#### 6. 코드베이스 방황

AI가 README, package.json, 설정 파일 3개, 관계없는 모듈 2개를 열고 나서야 코드 한 줄을 작성. 연속 5회 읽기, 편집 없음, 결정 없음. 낭비된 턴 $0.085 + 컨텍스트 세금 $0.027 = **에피소드당 $0.112.** 평균: **세션당 $0.09.**

수정: 먼저 grep이나 glob, 관련 있는 것만 읽기, 턴당 여러 읽기를 배치 처리. **세션당 약 $0.07 절약.**

#### 7. 배치되지 않은 편집

파일 A 편집, 다음 B, 다음 C — 세 턴. 병렬 편집으로 한 턴이면 충분. 추가 2턴 x $0.017 = **인스턴스당 $0.034.** 평균: **세션당 $0.058.**

수정: *"독립적인 도구 호출을 배치 처리."* **세션당 약 $0.05 절약.**

### Tier 3 — 꼬리 (낭비의 5-10%)

개별로는 작지만 합치면 큽니다.

#### 8. 파일 재읽기

같은 파일을 한 세션에서 두 번 읽기 — 내용이 이미 컨텍스트에 있는데 AI가 또 가져옴. **재읽기당 $0.019**, 파일은 평균 3-4회 재읽기. **세션당 $0.066.** 수정: *"이미 컨텍스트에 있음. 파일이 변경된 경우에만 재읽기."* **세션당 약 $0.05 절약.**

#### 9. Sleep/Poll 루프

`sleep 5 && check_status`를 3-5회 반복. 각 poll = 백그라운드 프로세스 완료 확인을 위한 전체 컨텍스트 재읽기. 4회 poll x $0.017 = **에피소드당 $0.068**, **세션당 $0.043.** 수정: *"--wait 또는 run_in_background 사용."* **세션당 약 $0.034 절약.**

#### 10. 실패 재시도

명령이 실패하고 AI가 같은 명령을 그대로 재실행. 에러 출력이 이제 컨텍스트에 두 번 존재. **재시도당 $0.019**, **세션당 $0.080.** 수정: 핑퐁과 같음 — *"멈추고, 에러를 읽고, 다른 것을 시도."*

#### 11. 스키마 조회

AI가 자체 도구 정의를 조회 — 이미 가지고 있는 정보. 2K+ token이 쓸데없이 추가. **세션당 $0.023.** "도구 호출 없는 턴 금지" 규칙으로 해결. **세션당 약 $0.02 절약.**

#### 12. Git 의식

`git add` → `git status` → `git commit` → `git push`. 네 턴. `git add -A && git commit -m "msg" && git push`는 한 턴. **인스턴스당 $0.044** 하지만 생각보다 드물게 발생 — **세션당 $0.003.** 수정: `&&`로 체인.

### Tier 4 — 상시 작동 에이전트

다른 비용 모델. OpenClaw 같은 에이전트는 주기적으로 깨어나며, 낭비는 세션이 아닌 일 단위로 측정.

#### 13. 유휴 하트비트

에이전트가 5분마다 깨어나 전체 워크스페이스를 다시 읽고, 아무것도 없고, 다시 잠듦. 하루 288회 기상, 약 97% 유휴. 280회 유휴 기상 x $0.04 = **하루 $11.20 (월 $336)** 아무것도 하지 않으면서.

수정: *"최소 30분 하트비트. 대기 중인 트리거가 없으면 건너뛰기."* 하루 약 48회로 감소. **하루 $8-10 절약 (월 $240-300).**

#### 14. 워크스페이스 파일 비대화

35,000 token의 성격 파일(SOUL.md, AGENTS.md 등)이 매번 기상 시 다시 읽힙니다. 튜토리얼, 코칭, 철학 — 사람을 위해 작성되었지 태스크를 실행하는 AI를 위한 것이 아닙니다. **하루 $5.76 (월 $173)** 설정 파일만으로.

vibecheck이 압축: 35K → 12-15K token. 같은 동작 규칙, 사람 대상 채움 내용 제거. **하루 $3-4 절약 (월 $90-120).**

#### 15. 메모리 축적

세션 기록이 무한히 증가. 100개 이상의 메모리 항목이 매번 기상 시 다시 읽히며, 몇 주 전의 더 이상 중요하지 않은 것도 포함. **하루 $3.17 (월 $95)** 오래된 메모리에.

수정: *"50개 항목 이후 아카이브, 요약, 새로 시작."* **하루 $2-3 절약 (월 $60-90).**

## 최적화 도구 모음

vibecheck은 문제를 지적할 뿐 아니라 수정합니다.

### 인스트럭션 파일 압축

인스트럭션 파일(`CLAUDE.md`, `AGENTS.md`, `Memory.md`, 도구마다 부르는 이름이 다릅니다)은 매 턴 읽힙니다. 모든 작업에 부과되는 고정 세금입니다. 비대한 인스트럭션 파일은 도시의 모든 도로에 설치된 통행료입니다.

vibecheck의 4단계 무손실 압축기 — 23가지 기법, "무손실"은 문자 그대로 사실이 하나도 삭제되지 않는다는 뜻. 같은 정보, 더 적은 token.

| 단계 | 내용 | 절감률 |
|---|---|---|
| **1단계 — 기계적** | Markdown 포맷 제거, 테이블을 간결한 형태로 변환, 불릿 병합 | 10-15% |
| **2단계 — 사실 보존** | 중복 사실 제거, 코드 예제 압축, 장황한 설명 축소 | 15-25% |
| **3단계 — 고충실도** | 사람에게는 필요하지만 AI에게는 불필요한 튜토리얼과 코칭 텍스트 제거 | 10-15% |
| **4단계 — 전보식** | AI 전용 파일을 위한 완전한 약칭 재작성. 고밀도, 압축 — 명시적 허가 필요 | 15-25% |

10,000 token 인스트럭션 파일을 6,000으로 압축하면 세션당 $0.044 절약. 하루 10세션: **압축만으로 $0.44/일 ($13/월).**

### 출력 억제

출력 token은 입력의 5배 비용 (Sonnet 4.6에서 $15 vs. $3/백만). AI가 요청하지 않은 전체 코드 블록이나 diff를 출력? 비쌉니다. vibecheck 추가 규칙: *"요청하지 않는 한 코드나 diff를 출력하지 마라."* **세션당 약 $0.047 절약.**

### 비용 모니터링

`/vibecheck monitor`는 세션 프로필의 스냅샷을 찍고 이후 실행에서 베이스라인과 비교합니다. 새 인스트럭션 파일이 장황함을 도입했나요? 다른 프로젝트, 다른 습관? 모니터가 비용 회귀를 누적 전에 잡아냅니다.
