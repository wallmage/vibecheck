- AI가 "네, 고쳐드릴게요"라고 말한 후 실제 작업을 다음 턴에 합니다. 그 narration 턴: **$0.031 낭비.** 세션당 5번이면: **$0.165 손실.**
- 대화가 20턴에서 분리되는 대신 40턴까지 이어집니다. 그 모든 대화 기록을 다시 읽는 추가 비용: **$0.67 낭비.**
- `git add`, 그다음 `git commit`, 그다음 `git push` — 하나의 체인 명령 대신 세 턴: **$0.098 낭비.**

이것이 vibecheck이 잡아내는 15가지 낭비 패턴 중 3가지입니다. 아래에 금액, 문제점, 수정 방법이 모두 설명되어 있습니다.

Claude, GPT, Gemini, DeepSeek, Qwen, Kimi, GLM, MiniMax와 호환됩니다. 24개 이상의 코딩 도구를 지원합니다. 로컬에서 실행되어 데이터가 사용자 기기에 머뭅니다.

### skill이란 무엇인가요?

AI를 위한 레시피 카드입니다. 아무것도 수정하거나 설치하지 않습니다. 단순히 "낭비를 찾고 수정하는 방법"을 설명하는 텍스트 파일입니다. 원할 때 언제든지 삭제할 수 있습니다.

### 코딩 도구 vs 채팅 도구

**코딩 도구** (Claude Code, Codex, Cursor, OpenClaw, Copilot, Windsurf, TRAE, Qoder 등)는 사용자 기기에서 실행됩니다 — vibecheck이 도구를 자동 감지하고 세션 로그를 스캔합니다.

**채팅 도구** (Cowork, 브라우저 기반)는 샌드박스에서 실행됩니다. vibecheck은 여전히 instruction 파일을 최적화하거나 (이것이 대부분의 효과입니다), 터미널 명령 하나로 14일치 로그를 복사해 전체 스캔을 진행할 수 있습니다.

### 권한

vibecheck은 instruction 파일 (CLAUDE.md, .cursorrules 등)을 읽고 편집합니다. 모든 변경 전에 확인을 요청합니다.

## 개인정보 보호

데이터가 사용자 기기 밖으로 나가지 않습니다. 서버, API, 텔레메트리 없음. 오픈 소스.

## 턴이 비용으로 이어지는 원리

매 턴마다 AI는 전체 대화를 다시 읽습니다 — 시스템 프롬프트, instruction 파일, 이전의 모든 메시지, 모든 도구 출력 — 그런 다음 응답을 생성합니다.

**턴 비용 = 입력 token × 입력 요금 + 출력 token × 출력 요금**

초반 턴은 저렴합니다 (작은 context). 후반 턴은 비쌉니다 (이전의 모든 내용을 다시 읽기 때문). 그래서 낭비는 복리처럼 불어납니다 — 낭비된 턴이 미래의 모든 턴을 더 비싸게 만드는데, 낭비된 내용이 context에 남아 있기 때문입니다.

프롬프트 캐싱은 이미 본 content의 입력 비용을 10배 절감합니다. 대부분의 도구는 자동으로 사용합니다.

**구독 사용자:** API 가격이 직접 보이지 않지만, 낭비는 메시지 할당량을 더 빠르게 소진시킵니다. Claude Pro ($20/월)는 API 기준 약 $200의 가치를 커버합니다. Max ($200/월)는 약 $4,000을 커버합니다.

## 15가지 낭비 패턴

### Tier 1 — 빅 3 (낭비의 60-70%)

#### 1. Idle Narration

**정의.** AI가 "네, 고쳐드릴게요" 또는 "먼저 파일을 읽겠습니다"라고 말한 후 — 실제 작업을 다음 턴에 합니다. 그 narration 턴은 아무것도 하지 않습니다: 도구 호출 없음, 코드 없음, 파일 읽기 없음.

**낭비.** 각 narration 턴은 **$0.031** 의 비용이 발생합니다 (context 재독 + 약 500 tokens의 상태 텍스트). 대부분의 세션에서 이런 경우가 5번 발생합니다: **$0.165/세션 낭비** — 청구 금액의 17%가 아무것도 만들지 않습니다. 하루 10세션이면: **$1.65/일 ($50/월)** 이 narration에만 쓰입니다.

**수정.** vibecheck이 추가합니다: *"도구 호출 없는 턴은 없습니다. Narration 없음. 같은 턴에 생각하고 행동하세요."* Narration을 완전히 없앱니다. **$0.15-0.18/세션 절약.**

#### 2. Context Rot

**정의.** 대화가 길어질수록 점점 더 비싸집니다. 50번째 턴은 이전 49개의 턴 전체를 다시 읽습니다. 총 세션 비용은 길이에 따라 이차함수적으로 증가합니다.

**낭비.** 40턴 세션 1번: **$1.89.** 20턴 세션 2번 (동일한 작업): **$1.22.** 그 차이 — **$0.67** — 는 아무것도 사지 않습니다. 100턴의 경우: 세션 1번의 비용은 **$5.62** 대 25턴 세션 4번의 **$3.84.** 분리하지 않아서 **$1.78 낭비.**

**수정.** 가르칩니다: *"관련 없는 작업 사이에 /clear 또는 /compact를 사용하세요. 새 대화를 시작하세요."* **긴 세션 습관이 있는 사용자에게 $0.30-0.70/세션 절약.**

#### 3. Ping-Pong Debugging

**정의.** 수정, 실패, 재시도, 또 실패. 각 실패 시도는 오류 출력을 context에 추가합니다 (사이클당 ~4K tokens), 이후 모든 턴에서 다시 읽힙니다.

**낭비.** 3번의 실패 사이클: 6 추가 턴 ($0.252) + 12K tokens의 죽은 오류들 ($0.036 context 세금). **총계: 에피소드당 ~$0.29.** 세션의 약 1/3에서 발생합니다. **가중 평균: ~$0.10/세션.**

**수정.** 추가합니다: *"같은 파일에서 2번 수정 실패 후: 멈추고, 오류를 완전히 다시 읽고, 생각하고, 단일 목표 수정을 합니다."* **에피소드당 ~$0.20 절약.**

### Tier 2 — 턴 밀도 (낭비의 15-20%)

#### 4. Verbose Tool Output

**정의.** 빌드/테스트 명령이 500줄 (~5K tokens)을 대화에 덤프합니다. 그 token들은 이후 모든 턴에서 다시 읽힙니다.

**낭비.** 5K tokens × 12 남은 턴 × $0.30/1M = **인스턴스당 $0.018** context 세금. 세션당 2-3번 발생합니다. 캐싱 없이: **인스턴스당 $0.180** — 10배 더 나쁩니다. **총계: $0.04-0.05/세션.**

**수정.** 추가합니다: *"빌드/테스트 출력을 /tmp/로 파이프하고, --quiet 플래그를 사용하고, 최대 tail -50."* **$0.03-0.05/세션 절약.**

#### 5. Unchained Commands

**정의.** 한 턴에 `npm install`, 다음 턴에 `npm run build`. `&&`로 체인하면 하나의 턴으로 처리 가능한데 context를 두 번 다시 읽습니다.

**낭비.** 분리할 때마다: **$0.023.** 일반적인 세션에는 3-4번의 분리가 있습니다. **총계: $0.07-0.09/세션.**

**수정.** 추가합니다: *"안전한 경우 `&&`로 명령어를 체인하세요."* **$0.06-0.08/세션 절약.**

#### 6. Codebase Wandering

**정의.** AI가 실제 작업을 하기 전에 파일들을 차례로 엽니다 — README, package.json, 설정 파일들. 첫 번째 편집 전에 5번 이상 연속 읽기.

**낭비.** 5번의 불필요한 읽기: 턴에서 $0.190 + context 세금 $0.027 = **에피소드당 $0.217.** 세션의 약 25%에서 발생합니다. **가중 평균: ~$0.054/세션.**

**수정.** 목표 검색 (먼저 grep/glob), 턴당 여러 읽기 배치 처리를 권장합니다. **에피소드당 ~$0.15 절약.**

#### 7. Unbatched Edits

**정의.** 파일 A 편집, 그다음 B, 그다음 C — 병렬 편집 하나의 턴으로 처리할 수 있는데 세 턴.

**낭비.** 2 추가 턴 × $0.038 = **인스턴스당 $0.076.** 세션의 약 60%에서 발생합니다. **가중 평균: ~$0.046/세션.**

**수정.** 추가합니다: *"독립적인 도구 호출을 배치 처리하세요 (턴당 여러 읽기/편집)."* **~$0.04/세션 절약.**

### Tier 3 — 꼬리 (낭비의 5-10%)

#### 8. File Re-reads

**정의.** 한 세션에서 같은 파일을 두 번 읽습니다. 첫 번째 읽기 후 content가 이미 context에 있습니다.

**낭비.** 1 낭비된 턴 + 중복 content = **재읽기당 $0.043.** 세션당 1-2번. **가중 평균: ~$0.039/세션.**

**수정.** 추가합니다: *"첫 번째 읽기 후 content가 context에 있습니다. 파일이 변경된 경우에만 재읽기."* **~$0.03/세션 절약.**

#### 9. Sleep/Poll Loops

**정의.** `sleep 5 && check_status`, 3-5번 반복. 각 폴링은 전체 context를 다시 읽습니다.

**낭비.** 4번 폴링 × $0.038 = **에피소드당 $0.152.** 세션의 약 20%에서 발생합니다. **가중 평균: ~$0.030/세션.**

**수정.** 추가합니다: *"--wait 플래그 또는 run_in_background를 사용하세요."* **에피소드당 ~$0.12 절약.**

#### 10. Failed Retries

**정의.** 명령이 실패하면 AI가 정확히 같은 명령을 다시 실행합니다. 오류 출력이 이제 context에 두 번 있습니다.

**낭비.** **재시도당 $0.042.** 세션의 약 30%에서 발생합니다. **가중 평균: ~$0.013/세션.**

**수정.** 핑퐁과 동일한 규칙: *"멈추고, 오류를 다시 읽고, 생각하고, 단일 목표 수정을 합니다."*

#### 11. Schema Lookups

**정의.** AI가 이미 가지고 있는 정보인 자신의 도구 정의를 조회합니다. context에 2K+ tokens을 추가합니다.

**낭비.** **조회당 $0.052.** 세션의 약 40%에서 발생합니다. **가중 평균: ~$0.021/세션.**

**수정.** "도구 호출 없는 턴은 없습니다"가 탐색 턴을 억제합니다. **~$0.02/세션 절약.**

#### 12. Git Ceremony

**정의.** `git add` → `git status` → `git commit` → `git push`, 네 턴. `git add -A && git commit -m "msg" && git push`가 하나입니다.

**낭비.** 3 추가 턴 + git 출력 = **인스턴스당 $0.098.** 세션의 약 70%에서 발생합니다. **가중 평균: ~$0.069/세션.**

**수정.** 추가합니다: *"`&&`로 git 명령어를 체인하세요."* **~$0.06/세션 절약.**

### Tier 4 — 상시 구동 에이전트 (OpenClaw 등)

다른 비용 모델: 깨어남당 비용 × 하루 깨어남 횟수.

#### 13. Idle Heartbeats

**정의.** 에이전트가 5분마다 깨어나서 전체 워크스페이스를 다시 읽고 아무것도 찾지 못합니다. 하루 288번 깨어남, 약 97%가 유휴 상태.

**낭비.** 하루 280번 유휴 깨어남 × $0.04 = **$11.20/일 ($336/월)** 아무것도 하지 않음.

**수정.** *"최소 30분 하트비트. 트리거가 없으면 건너뜀."* 하루 약 48번으로 줄어듭니다. **$8-10/일 ($240-300/월) 절약.**

#### 14. Workspace File Bloat

**정의.** 35K tokens의 개인 설정 파일들 (SOUL.md, AGENTS.md)이 매 깨어남마다 다시 읽힙니다. AI에게 필요한 것은 행동 규칙뿐 — 튜토리얼과 코칭은 사람을 위한 것입니다.

**낭비.** **$5.76/일 ($173/월)** 설정 파일만 읽는 데 소비.

**수정.** 워크스페이스 파일을 압축합니다: 35K → 12-15K tokens. **$3-4/일 ($90-120/월) 절약.**

#### 15. Memory Accumulation

**정의.** 세션 기록이 정리 없이 증가합니다. 100개 이상의 항목이 매 깨어남마다 다시 읽힙니다.

**낭비.** **$3.17/일 ($95/월)** 오래된 메모리를 읽는 데 소비.

**수정.** *"50턴 후 아카이브, 요약, 새로 시작."* **$2-3/일 ($60-90/월) 절약.**

## 추가: 최적화 도구

### Instruction 파일 압축

Instruction 파일은 매 턴마다 읽힙니다 — 작업과 관계없이 고정으로 내는 세금입니다. vibecheck은 23가지 기법의 4패스 무손실 압축기를 포함하여 파일 크기를 25-50% 줄입니다:

- **패스 1 (기계적):** 마크다운 제거, 테이블 변환, 글머리 기호 병합. ~10-15%.
- **패스 2 (사실 보존):** 중복 사실 제거, 코드 예시 압축. ~15-25%.
- **패스 3 (고충실도):** 사람에게는 필요하지만 AI에게는 불필요한 튜토리얼 및 코칭 텍스트 제거. ~10-15%.
- **패스 4 (전보체):** AI 전용 파일에 대한 완전한 약어 재작성. ~15-25% (허가 시에만).

10K token 파일을 6K로 압축하면 세션당 $0.057 절약. 하루 10세션이면: **$0.57/일 ($17/월).**

### 출력 억제

출력 token은 입력보다 5배 비쌉니다 (Sonnet 기준 $15 vs $3/MTok). AI가 요청하지 않은 전체 코드 블록과 diff를 보여주면 **~$0.047/세션 낭비됩니다.** vibecheck이 추가합니다: *"요청하지 않으면 코드/diff 없음."*

### 비용 모니터링

`/vibecheck monitor`는 세션 프로필의 스냅샷을 찍고 이후 실행 시 기준값과 비교합니다. 비용이 발생하기 전에 회귀를 감지합니다.
