### スキルとは何ですか？

AIへのレシピカードです。何も変更しないし、何もインストールしません。「無駄を見つけて修正する方法」を書いたテキストファイルです。いつでも削除できます。

### コーディングツール vs チャットツール

**コーディングツール**（Claude Code、Codex、Cursor、OpenClaw、Copilot、Windsurf、TRAE、Qoder など）はあなたのマシン上で動作します — vibecheck がツールを自動検出してセッションログをスキャンします。

**チャットツール**（Cowork、ブラウザベース）はサンドボックス内で動作します。vibecheck は引き続きインストラクションファイルを最適化します（効果の大部分はここです）。フルスキャンを行うには、1つのターミナルコマンドを貼り付けて14日分のログをコピーするだけです。

### 権限

vibecheck はインストラクションファイル（CLAUDE.md、.cursorrules など）を読み取り、編集します。変更の前に必ず確認を求めます。

## ターンがお金を消費する仕組み

毎ターン、AIはシステムプロンプト、インストラクションファイル、すべての過去メッセージ、すべてのツール出力など、会話全体を読み返してからレスポンスを生成します。

**ターンコスト = 入力 token 数 × 入力レート + 出力 token 数 × 出力レート**

序盤のターンは安い（context が小さい）。終盤のターンは高い（それまでの全内容を読み返す）。だから無駄は複利で増える — 無駄なターンがあると、その内容が context に残り続けるため、以降のすべてのターンがより高くなります。

Prompt caching は既に処理済みのコンテンツの入力コストを10分の1に削減します。ほとんどのツールは自動的に使用しています。

**サブスクリプションユーザーの方へ：** APIの価格は直接見えませんが、無駄が多いとメッセージ枠を早く消費します。Claude Pro（月$20）は約$200のAPI価値に相当。Max（月$200）は約$4,000に相当。

## 15の無駄なパターン

### Tier 1 — ビッグ3（無駄の60〜70%）

#### 1. アイドルナレーション

**内容：** AIが「わかりました、修正します」や「まずファイルを読みます」と言ってから、次のターンで実際の作業を行う。ナレーションターンは何もしていない：ツール呼び出しなし、コードなし、ファイル読み取りなし。

**無駄：** 428セッションの実データ：**$1.03/セッション — 無駄の30%。** 1日10セッションなら：**$10.30/日（$309/月）** のナレーションだけで消えます。

**修正：** vibecheck が追加：*「ツール呼び出しのないターンは禁止。ナレーション禁止。考えと行動を同じターンで。」* ナレーションを完全に排除。**約$0.88/セッションの節約。**

#### 2. Context の劣化

**内容：** 長い会話は徐々に高くなります。50ターン目は過去49ターン全体を読み返します。セッションの総コストは長さの二乗で増加します。

**無駄：** 40ターンのセッション1回：**$0.70。** 20ターンのセッション2回（同じ作業）：**$0.60。** その差 **$0.10** は何も生まない。100ターンなら：1セッションのコストは **$2.53**、25ターンのセッション4回なら **$1.64。** 分割しないことで **$0.89の無駄。** 428セッションの実データ：**$0.46/セッション — 無駄の13%。**

**修正：** *「関係のないタスクの間は /clear または /compact を使う。新しい会話を始める。」* **約$0.37/セッションの節約。**

#### 3. ピンポンデバッグ

**内容：** 修正、壊れる、リトライ、また壊れる。失敗した試みのたびにエラー出力が context に追加され（サイクルあたり約4K tokens）、以降のすべてのターンで読み返されます。

**無駄：** 3回の失敗サイクル：6ターン分の余分（$0.102）+ 12K tokens の無効なエラー（$0.036の context 税）。**合計：エピソードあたり約$0.14。** 発生頻度約10%。428セッションの実データ：**$0.015/セッション。**

**修正：** *「同じファイルで2回修正に失敗したら：止まり、エラーを完全に読み、考えて、的を絞った1回の修正を行う。」* **約$0.01/セッションの節約。**

### Tier 2 — ターン密度（無駄の15〜20%）

#### 4. 冗長なツール出力

**内容：** ビルド/テストコマンドが500行（約5K tokens）を会話に出力する。これらの tokens は以降のすべてのターンで読み返されます。

**無駄：** 5K tokens × 残り12ターン × $0.30/1M = **インスタンスあたり $0.018** の context 税。セッションに2〜3回発生。caching なしでは：**インスタンスあたり $0.180** — 10倍悪化。428セッションの実データ：**$1.05/セッション — 無駄の31%。**

**修正：** *「ビルド/テスト出力を /tmp/ にパイプし、--quiet フラグを使い、tail -50 を最大にする。」* **約$0.89/セッションの節約。**

#### 5. コマンドの非チェーン化

**内容：** 1ターンで `npm install`、次のターンで `npm run build`。`&&` でチェーンすれば1ターンで済むのに context を2回読み返す。

**無駄：** 分割1回あたり：**$0.010。** 典型的なセッションに3〜4回の分割。428セッションの実データ：**$0.14/セッション。**

**修正：** *「安全なときは `&&` でコマンドをチェーンする。」* **約$0.11/セッションの節約。**

#### 6. コードベースの徘徊

**内容：** AIが README、package.json、設定ファイルと次々にファイルを開き、実際の作業前に何もしない。最初の編集前に5回以上連続して読み取り。

**無駄：** 不要な読み取り5回：ターン分の $0.085 + context 税 $0.027 = **エピソードあたり $0.112。** 428セッションの実データ：**$0.09/セッション。**

**修正：** 的を絞った検索（まず grep/glob を使う）、1ターンに複数の読み取りをまとめることを推奨。**約$0.07/セッションの節約。**

#### 7. 非バッチ編集

**内容：** ファイルAを編集、次にB、次にC — 1ターンで並行編集できるのに3ターンかける。

**無駄：** 余分な2ターン × $0.017 = **インスタンスあたり $0.034。** 428セッションの実データ：**$0.058/セッション。**

**修正：** *「独立したツール呼び出しをバッチ処理する（1ターンに複数の読み取り/編集）。」* **約$0.05/セッションの節約。**

### Tier 3 — テール（無駄の5〜10%）

#### 8. ファイルの再読み取り

**内容：** 1セッション内で同じファイルを2回読み取る。最初の読み取り後、内容はすでに context にある。

**無駄：** 1回の無駄なターン + 重複コンテンツ = **再読み取りあたり $0.019。** 428セッションの実データ：**$0.066/セッション。**

**修正：** *「最初の読み取り後、内容は context にある。ファイルが変更された場合のみ再読み取りする。」* **約$0.05/セッションの節約。**

#### 9. Sleep/Poll ループ

**内容：** `sleep 5 && check_status` を3〜5回繰り返す。各 poll で全 context を読み返す。

**無駄：** poll 4回 × $0.017 = **エピソードあたり $0.068。** 428セッションの実データ：**$0.043/セッション。**

**修正：** *「--wait フラグまたは run_in_background を使う。」* **約$0.034/セッションの節約。**

#### 10. 失敗リトライ

**内容：** コマンドが失敗し、AIがまったく同じコマンドを再実行する。エラー出力が context に2回存在することになる。

**無駄：** **リトライあたり $0.019。** 428セッションの実データ：**$0.080/セッション。**

**修正：** ピンポンと同じルール：*「止まり、エラーを読み、考えて、的を絞った1回の修正。」*

#### 11. スキーマ参照

**内容：** AIが自身のツール定義を参照する — すでに持っている情報。context に2K以上の tokens が追加される。

**無駄：** **参照あたり $0.023。** 428セッションの実データ：**$0.023/セッション。**

**修正：** 「ツール呼び出しのないターンは禁止」で発見ターンを抑制。**約$0.02/セッションの節約。**

#### 12. Git の儀式

**内容：** `git add` → `git status` → `git commit` → `git push`、4ターン。`git add -A && git commit -m "msg" && git push` で1ターンで済む。

**無駄：** 余分な3ターン + git 出力 = **インスタンスあたり $0.044。** 428セッションの実データ：**$0.003/セッション。**

**修正：** *「git コマンドを `&&` でチェーンする。」* **約$0.003/セッションの節約。**

### Tier 4 — 常時稼働エージェント（OpenClaw など）

コストモデルが異なる：1回の起動コスト × 1日あたりの起動回数。

#### 13. アイドルハートビート

**内容：** エージェントが5分ごとに起動し、ワークスペース全体を読み返して何も見つけない。1日288回起動、約97%がアイドル。

**無駄：** 1日あたりアイドル起動280回 × $0.04 = **何もしないのに $11.20/日（$336/月）。**

**修正：** *「最低30分のハートビート間隔。トリガーがなければスキップ。」* 1日約48回の起動に削減。**$8〜10/日（$240〜300/月）の節約。**

#### 14. ワークスペースファイルの肥大化

**内容：** 35K tokens のペルソナファイル（SOUL.md、AGENTS.md）を毎回の起動時に読み返す。AIに必要なのは行動ルールだけ — チュートリアルやコーチング文章は人間向け。

**無駄：** 設定ファイルを読むだけで **$5.76/日（$173/月）。**

**修正：** ワークスペースファイルを圧縮：35K → 12〜15K tokens。**$3〜4/日（$90〜120/月）の節約。**

#### 15. メモリの蓄積

**内容：** セッション履歴が刈り込みなしに成長し続ける。100件以上のエントリーを毎回の起動時に読み返す。

**無駄：** 古いメモリを読み続けるだけで **$3.17/日（$95/月）。**

**修正：** *「50ターン後にアーカイブし、要約して、新鮮な状態から始める。」* **$2〜3/日（$60〜90/月）の節約。**

## プラス：最適化ツール

### インストラクションファイル圧縮

インストラクションファイルは毎ターン読み取られます — タスクに関係なく支払う固定のコスト。vibecheck には4パスの無損失コンプレッサー（23のテクニック）が含まれており、ファイルサイズを25〜50%削減します：

- **Pass 1（機械的処理）：** Markdown を除去し、テーブルを変換し、箇条書きをまとめる。約10〜15%。
- **Pass 2（事実保存）：** 重複した事実を排除し、コード例を圧縮する。約15〜25%。
- **Pass 3（高忠実度）：** 人間には必要だがAIには不要なチュートリアルやコーチングテキストを削除する。約10〜15%。
- **Pass 4（電報式）：** AIのみのファイル向け完全略記リライト。約15〜25%（許可を得た場合のみ）。

10K token のファイルを6Kに圧縮すると $0.044/セッションの節約。1日10セッションなら：**$0.44/日（$13/月）。**

### 出力抑制

出力 tokens は入力の5倍のコストがかかります（Sonnet では $3/MTok に対して $15/MTok）。頼んでもいないのにAIが完全なコードブロックや差分を表示すると **約$0.047/セッションの無駄。** vibecheck が追加：*「求められない限りコード/差分を表示しない。」*

### コスト監視

`/vibecheck monitor` がセッションプロファイルのスナップショットを取得し、次回の実行時にベースラインと比較します。コストがかかる前にリグレッションを検出します。

## 対応ツール

24以上のツールに対応。

- **フルセッションスキャン：** Claude Code、Codex、Cursor、OpenClaw、GitHub Copilot、Windsurf、TRAE、Qoder、CodeBuddy、WorkBuddy、Google Antigravity
- **検出 + インストラクション最適化：** Cline、Roo Code、Kilo Code、Aider、Gemini CLI、OpenCode、Augment、Kimi Code、MarsCode、Tongyi Lingma、Baidu Comate、CodeGeeX、DevChat、MiniMax

すべての LLM に対応：Claude Opus/Sonnet 4.6、GPT-5.4、Gemini 3.1 Pro、DeepSeek V3.2、Qwen 3.6、Kimi K2.5、GLM-5、MiniMax M2.7、その他40以上。

macOS、Windows、Linux、SSH 経由の iPad。Python 3.8+、依存関係なし。

<summary>方法論</summary>

すべてのコスト見積もりは上記の基準シナリオを使用しています。主な前提条件：

- **95% の prompt cache ヒット率** — 高速なコーディングセッションの典型値。低速なセッションではコストが高くなります。
- **1セッションあたり25の生産的なターン** — 無駄の多いセッションでは、ナレーション、リトライ、非チェーンコマンドにより8〜12ターンの余分が発生します。
- **ターンあたり600 tokens の増加** — 冗長なセッションでは1,000〜2,000に達することがあります。
- **実効入力レート：$0.435/1M** — 95%キャッシュ済み（$0.30）+ 5%未キャッシュ（$3.00）のブレンド。
- **Context 税レート：$0.30/1M** — 永続的な context 追加に対するキャッシュ済み入力レート。

見積もりは保守的です。長いセッション、大きなインストラクションファイル、または重いデバッグを行うユーザーの実際の節約額は予測を超えることがあります。
