毎日午後になると Claude の使用枠を使い切ってしまい、原因がわからなかった。調べてみると、AI コーディングセッションの 70% が無駄だった——AI がこれからやることをいちいち説明したり、1回で済むコマンドを3ターンに分けたり、古いコンテキストが積み重なって毎ターン再読み込みされていた。

vibecheck はその無駄を見つける。24以上のコーディングツールの実際のセッションログを読み取り、15種類の具体的なパターンに金額を付けて、修正する。すべてローカルで実行。アップロードなし、テレメトリなし、サーバーなし。

私の場合：月額費用が $2,816 から $422 に。**85% 削減。**

## インストール方法

AI コーディングツールにこれを貼り付けて Enter を押す：

> Help me install this skill: https://github.com/wallmage/vibecheck

以上。AI が skill を読み込み、スキャンの準備完了。

### 「skill」とは？

AI に新しいことを教える純粋なテキストファイル。バイナリなし、バックグラウンドプロセスなし、システム変更なし。vibecheck の skill ファイルは「無駄の見つけ方と直し方」を記述したもの。フォルダを削除すれば消える。

### コーディングツール vs. チャットツール

**コーディングツール**（Claude Code、Codex、Cursor、OpenClaw、Copilot、Windsurf、TRAE、Qoder など）はローカルで動作し、セッションログを残す。vibecheck はインストール済みのツールを自動検出し、それらのログを直接スキャンする。

**チャットツール**（Cowork、ブラウザ版 Claude）はサンドボックスで動作し、ローカルログがない。vibecheck はインストラクションファイルの最適化には対応——実際、節約の大部分はそこから生まれる。ターミナルコマンド1つを貼り付ければ14日分のログをエクスポートしてフルスキャンもできる。

### 権限

vibecheck はローカルのセッションログを読み取り、インストラクションファイル（`CLAUDE.md`、`AGENTS.md`、`Memory.md`）とマシン全体のツール設定を検査する。ツールにグローバル設定ファイル（全プロジェクト共通の1ファイル）がある場合、オプティマイザはそこを最初に処理する。1箇所の修正で全体のコストが下がるから。何かを変更する前に必ず確認する。

## プライバシー

すべてのデータはあなたのマシン上に留まる。分析はローカルのセッションログをパースする Python スクリプト群。サーバーなし、API 呼び出しなし、アナリティクスなし。オープンソース——すべての行を確認できる。

## コマンド

| コマンド | 機能 |
|---|---|
| `/vibecheck scan` | マシン上で検出されたすべてのツールをスキャン。ヘルスマーカー、ランク付き統計、主な無駄パターン、最適化ロードマップを含む統合レポート |
| `/vibecheck explain` | 何も変更せずに無駄パターンを解説。純粋な学習用 |
| `/vibecheck compress` | 4パスのロスレスコンプレッサーでインストラクションファイルを 25-50% 圧縮 |
| `/vibecheck monitor` | ベースラインとの週次比較。コストの逆戻りを早期発見 |

スキャンは静かに進行：コンパクトな進捗インジケータの後、クリーンなサマリー。`Good` は測定された無駄が 10% 未満、`Waste` はそれ以上。生ログやツールの出力は問題が発生しない限り表示されない。

### セッションを新鮮に保つ

長い会話は短い会話より高くつく——新しいターンごとに過去のすべてのターンを再読み込みし、肥大化したコンテキストは AI の精度を下げ、やり取りが増える。

目安：セッションあたり 5-10 分の実作業、30-40 ターンを超えるとコンテキスト税が顕著になる。新しいセッションを始める際は、持続的なルールをインストラクションファイル（`CLAUDE.md`、`AGENTS.md`、`Memory.md`）に、プロジェクト背景を小さなローカルドキュメントに置く。新しいセッション = ゼロからのスタートではない。クリーンなコンテキストに、蓄積した知識がすべて残っている状態。

## AI のターンがお金になる仕組み

トークンエコノミクスを考えたことがない人向けの簡単な入門。AI の料金体系の知識は不要。

### 各ターンで何が起きているか

AI が応答するたびに、会話全体を最初から再読み込みする。システムプロンプト、インストラクションファイル、あなたが送ったすべてのメッセージ、AI のすべての応答、すべてのツール出力——ファイル内容、ターミナル結果、エラーログ——すべて。それから新しい応答を生成する。

**ターンコスト = 読み取り token 数 x 入力単価 + 生成 token 数 x 出力単価**

初期のターンは安い。ターン 1 は 5,000 token 程度を読むかもしれない。ターン 40 では、蓄積された会話 40,000 以上の token を再読み込み——過去のすべてのメッセージ、コードスニペット、エラートレース。その後半のターンは最初のターンの 8 倍のコスト。

無駄が高くつく理由は**複利効果**にある。無駄なターンは自身の token コストだけでなく、セッションの残り全体でコンテキストに残り続け、以後の全ターンで再読み込みされる。ターン 10 での不要な説明メッセージ1つが、セッション終了まであと 30 回再読み込みされる。

### プロンプトキャッシュは助けになるが、根本的な解決にはならない

多くのプロバイダーは既に見たコンテンツをキャッシュし、料金を 10 分の 1 にしている。実効入力コストは $3.00/100万 token から $0.30/100万に下がる。

助けにはなる。だが新しいコンテンツ——新しいツール出力、新しいエラーメッセージ、各新規 AI 応答——はキャッシュされる前に常にフルプライスで入る。そしてキャッシュ料金でも無駄は複利で積み上がる。

### サブスクリプションでも同じ痛みがある

サブスクリプションなら API 料金は関係ないと思うかもしれない。関係ある——感じ方が違うだけ。サブスクリプションは固定量のコンピュートを購入しており、無駄はそのプールを速く消費する。午後 3 時にレート制限されるのは、働きすぎではなく、多くの作業が無駄だったから。

Claude Pro ($20/月) は API 換算で約 $200 相当。Claude 20x Max ($200/月) は約 $4,000 相当。無駄が多い = 壁に当たるのが早い。

### 測定方法

$200/月の Claude 20x Max プランを使っていたが、使用枠が常に足りなくなった。気になって API 課金に切り替え、100 以上のデータポイントで実際の支出を追跡——すべてのコーディング作業を記録し、直後に使用量を確認。これでサブスクリプション価格と実際の token 価値の関係が分かった。

### 倍率

| プラン | 価格 | API 価値 | 倍率 | 5時間枠 | 週間合計 |
|---|---|---|---|---|---|
| Claude Pro | $20/月 | ~$200 | 10x | $6.67 | $46.67 |
| Claude 5x Max | $100/月 | ~$1,000 | 10x | $33.33 | $233.31 |
| Claude 20x Max | $200/月 | ~$4,000 | 20x | $133.33 | $933.31 |

20x Max だけが本当の倍率アップ——額面の 20 倍。下位プランは 10 倍。

### 実際の使用感

- **$20 Claude Pro** ——本格的な開発作業（機能実装、リサーチ、ドキュメント作成）では 1 時間以内に 5 時間枠を使い切る。週間容量 7 時間未満。プロ用途には厳しい。
- **$100 5x Max** ——約 4 時間で 5 時間枠に到達。週間合計 30-35 時間。通常利用なら十分。
- **$200 20x Max** ——週 80-100 時間以上働き、複数セッションを並行して走らせる人向け。

### Anthropic がサードパーティのサブスクリプション利用を制限した理由

額面の 10-20 倍では、サブスクリプション 1 ドルごとに API 料金をはるかに超えるコンピュートを購入することになる。サードパーティツールが API 換算速度でそれを消費すると、採算が合わなくなった。

### Codex という選択肢

$20 プランでは、Codex Plus は Claude Pro の約 **3 倍のコーディング使用量**を提供する。ChatGPT の会話——GPT-5.4 Extended Thinking やディープリサーチを含めても——Codex のコーディング枠にはカウントされない。つまり 3 倍のコーディング容量 + 無料の GPT-5.4 が付く。

**月 $20 の予算なら、Codex Plus は Claude Pro よりコーディング時間が多い。** それ以上の予算なら、Claude 5x と 20x が異なるバリュープロポジションを提供する。

## 15の無駄パターン

コスト影響の大きい順に整理。上位 3 つだけで全無駄の 60-70% を占める。

### Tier 1——三大パターン（無駄の 60-70%）

#### 1. アイドルナレーション

AI が「では修正します」「まずファイルを読みます」と言い——実際の作業は次のターンで行う。そのナレーションターンは何もしていない。ツール呼び出しなし、コードなし、ファイル読み取りなし。ただのアナウンス。

1回あたり約 **$0.017**——さらに悪いことに、300-500 token のステータステキストがコンテキストに残り、以降の全ターンで再読み込みされる。428 セッションの測定で：**セッションあたり $1.03 の無駄**、全無駄の 30%。1日 10 セッションなら、**ナレーションだけで月 $309。**

vibecheck のルール：*「ツール呼び出しのないターンは禁止。考えて行動を同じターンで。」* **セッションあたり約 $0.88 節約。**

#### 2. コンテキストの腐敗

セッションコストは線形ではなく二次関数的に増加する。ターン 50 は前の 49 ターンすべてを再読み込みする。

具体的な比較：40 ターンのセッション 1 回は **$0.70**。同じ作業を 20 ターンのセッション 2 回に分けると **$0.60**。この $0.10 の差は肥大した会話を維持するだけの無駄。実際のセッションは平均 74 ターン——**セッションあたり $0.46 の無駄**、全無駄の 13%。

vibecheck の教え：*「関係のない作業は別セッションで。長いスレッドではコンパクトに。」* **セッションあたり約 $0.37 節約。**

#### 3. ピンポンデバッグ

修正、壊れる、リトライ、また壊れる。失敗するたびに約 4,000 token のエラー出力がコンテキストに入り、その不要なテキストは以降の全ターンで再読み込みされる。3サイクル：6 余分なターン ($0.102) + 12K token の古いエラー ($0.036) = **1エピソードあたり約 $0.14**。セッションの約 10% で発生。**加重平均：セッションあたり $0.015。**

vibecheck のサーキットブレーカー：*「同じファイルで 2 回修正に失敗したら——止まって、エラー全体を再読し、考えて、的確に 1 回で修正。」* **セッションあたり約 $0.01 節約。**

### Tier 2——ターン密度（無駄の 15-20%）

1 ターンで済むことを 3 ターンかけてやる。

#### 4. 冗長なツール出力

ビルドやテストコマンドが 500 行（約 5,000 token）を会話に投入。その token はセッションの残り全ターンで再読み込みされる。5K token x 12 残りターン x キャッシュ料金 = **1回あたり $0.018**。キャッシュなしなら：**$0.180**——10 倍。

実測で最もコスト高のパターン。ビルドログ、npm 出力、テストダンプ——ほぼ全セッションに影響。**セッションあたり $1.05**、全無駄の 31%。

修正：*「出力を /tmp/ にパイプ。--quiet フラグを使用。tail -50 まで。」* **セッションあたり約 $0.89 節約。**

#### 5. チェーンされていないコマンド

`npm install` を 1 ターン。`npm run build` を次のターン。`npm install && npm run build` なら 1 回のコンテキスト読み込みで済むのに 2 回。1 分割あたり：**$0.010**。コマンド多用セッションで合計 **セッションあたり $0.14**。

修正：*「安全な場合は `&&` でチェーン。」* **セッションあたり約 $0.11 節約。**

#### 6. コードベースの彷徨

AI が README、package.json、設定ファイル 3 つ、無関係なモジュール 2 つを開いてから、やっとコードを 1 行書く。5 回連続の読み取り、編集なし、判断なし。無駄なターン $0.085 + コンテキスト税 $0.027 = **1エピソードあたり $0.112。** 平均：**セッションあたり $0.09。**

修正：まず grep か glob、関連するものだけ読む、1 ターンで複数の読み取りをバッチ処理。**セッションあたり約 $0.07 節約。**

#### 7. バッチ化されていない編集

ファイル A を編集、次に B、次に C——3 ターン。並行編集なら 1 ターン。余分な 2 ターン x $0.017 = **1回あたり $0.034。** 平均：**セッションあたり $0.058。**

修正：*「独立したツール呼び出しはバッチ処理。」* **セッションあたり約 $0.05 節約。**

### Tier 3——テール（無駄の 5-10%）

個別には小さいが積み重なる。

#### 8. ファイルの再読み込み

同じファイルを 1 セッションで 2 回読む——内容はすでにコンテキストにあるのに、AI がまた取得する。**再読み込み 1 回あたり $0.019**、ファイルは平均 3-4 回再読み込みされる。**セッションあたり $0.066。** 修正：*「すでにコンテキストにある。ファイルが変更された場合のみ再読み込み。」* **セッションあたり約 $0.05 節約。**

#### 9. Sleep/Poll ループ

`sleep 5 && check_status` を 3-5 回繰り返す。各 poll = バックグラウンドプロセスの完了確認のためにコンテキスト全体を再読み込み。4 回 poll x $0.017 = **1エピソードあたり $0.068**、**セッションあたり $0.043。** 修正：*「--wait か run_in_background を使う。」* **セッションあたり約 $0.034 節約。**

#### 10. 失敗リトライ

コマンドが失敗し、AI が同じコマンドをそのまま再実行。エラー出力がコンテキストに 2 回入る。**リトライ 1 回あたり $0.019**、**セッションあたり $0.080。** 修正：ピンポンデバッグと同じ——*「止まって、エラーを読んで、別のことを試す。」*

#### 11. スキーマ参照

AI が自身のツール定義を参照——すでに持っている情報。2K+ token が無駄に追加される。**セッションあたり $0.023。**「ツール呼び出しのないターンは禁止」ルールで解決。**セッションあたり約 $0.02 節約。**

#### 12. Git セレモニー

`git add` → `git status` → `git commit` → `git push`。4 ターン。`git add -A && git commit -m "msg" && git push` なら 1 ターン。**1回あたり $0.044** だが想像より稀——**セッションあたり $0.003。** 修正：`&&` でチェーン。

### Tier 4——常時稼働エージェント

異なるコストモデル。OpenClaw のようなエージェントは定期的に起動し、無駄はセッション単位ではなく日単位で測定。

#### 13. アイドルハートビート

エージェントが 5 分ごとに起動し、ワークスペース全体を再読み込みし、何もなく、スリープに戻る。1日 288 回の起動、約 97% がアイドル。280 回のアイドル起動 x $0.04 = **1日 $11.20（月 $336）** 何もせずに。

修正：*「最低 30 分のハートビート。保留中のトリガーがなければスキップ。」* 1日約 48 回に削減。**1日 $8-10 節約（月 $240-300）。**

#### 14. ワークスペースファイルの肥大化

35,000 token のパーソナリティファイル（SOUL.md、AGENTS.md など）が起動のたびに再読み込みされる。チュートリアル、コーチング、哲学——人間向けに書かれており、タスク実行する AI 向けではない。**1日 $5.76（月 $173）** が設定ファイルだけで。

vibecheck で圧縮：35K → 12-15K token。同じ動作ルール、人間向けのフィラーなし。**1日 $3-4 節約（月 $90-120）。**

#### 15. メモリ蓄積

セッション履歴が際限なく成長。100 以上のメモリエントリが起動のたびに再読み込みされ、数週間前のもう関係ないものも含まれる。**1日 $3.17（月 $95）** が古いメモリに。

修正：*「50 エントリを超えたらアーカイブ、要約、リスタート。」* **1日 $2-3 節約（月 $60-90）。**

## 最適化ツールキット

vibecheck は問題を指摘するだけでなく、修正する。

### インストラクションファイル圧縮

インストラクションファイル（`CLAUDE.md`、`AGENTS.md`、`Memory.md`、ツールごとの呼び名は何であれ）は毎ターン読み込まれる。すべての作業にかかる固定税。肥大したインストラクションファイルは街中のすべての道路に設置された料金所。

vibecheck の 4 パスロスレスコンプレッサー——23 の手法、「ロスレス」は文字通り事実を一切削除しないということ。同じ情報、より少ない token。

| パス | 内容 | 削減率 |
|---|---|---|
| **パス 1——機械的** | Markdown フォーマット除去、テーブルをコンパクト化、箇条書きを統合 | 10-15% |
| **パス 2——事実保持** | 重複する事実を排除、コード例を圧縮、冗長な説明を折りたたむ | 15-25% |
| **パス 3——高忠実度** | 人間には必要だが AI には不要なチュートリアルやコーチングテキストを除去 | 10-15% |
| **パス 4——テレグラム** | AI 専用ファイル向けの完全な短縮書き換え。高密度・圧縮、ただし明示的な許可が必要 | 15-25% |

10,000 token のインストラクションファイルを 6,000 に圧縮すると、セッションあたり $0.044 節約。1日 10 セッションで：**圧縮だけで $0.44/日（$13/月）。**

### 出力抑制

出力 token は入力の 5 倍のコスト（Sonnet 4.6 で $15 vs. $3/100万）。AI が求めていないコードブロックや diff を出力する？高くつく。vibecheck の追加ルール：*「求められない限りコードや diff を出力しない。」* **セッションあたり約 $0.047 節約。**

### コストモニタリング

`/vibecheck monitor` はセッションプロファイルのスナップショットを取り、将来の実行でベースラインと比較。新しいインストラクションファイルが冗長性を持ち込んだ？プロジェクトや習慣が変わった？モニターはコストの逆戻りを蓄積する前にキャッチ。

<details>
<summary><strong>方法論</strong></summary>

すべてのコスト見積もりは上記の基準シナリオを使用。主な前提：

- **95% プロンプトキャッシュヒット率** ——高速なコーディングセッションの典型値。ターン間の間隔が長い場合、キャッシュヒット率が下がりコストが上がる。
- **セッションあたり 25 生産的ターン** ——無駄の多いセッションはナレーション、リトライ、チェーンされていないコマンドで 8-12 ターン追加。
- **ターンあたり 600 token の増分** ——冗長なセッションでは 1,000-2,000 token/ターンに達することも。
- **実効入力料金：$0.435/1M** ——95% キャッシュ ($0.30/1M) + 5% 非キャッシュ ($3.00/1M) のブレンド料金。
- **コンテキスト税率：$0.30/1M** ——永続的なコンテキスト追加に対するキャッシュ入力料金。

保守的な見積もり。実際の節約はこれを上回ることが多い——特に長いセッション、大きなインストラクションファイル、集中的なデバッグの場合。
</details>
