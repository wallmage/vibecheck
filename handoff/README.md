Switch chats frequently without losing the thread.

`handoff` is a sister skill to [vibecheck](https://github.com/wallmage/vibecheck). It gives you one copyable transfer block so you can move from one session to the next without dragging a bloated old conversation forward.

It works in:
- coding tools
- CLI tools
- GUI chat tools
- chat products like Claude chat / Cowork

## Why this exists

Long conversations get expensive and messy.

- each new turn re-reads more stale context
- overloaded context makes the model less sharp
- people avoid resetting because they do not want to lose decisions, momentum, and project background

`handoff` solves the continuity problem. You say `handoff`, `hand off`, or `start fresh`, copy the generated block, paste it into a new chat, and keep going.

## Install

Copy this into your AI tool:

```text
Help me install this skill: https://github.com/wallmage/handoff
```

## Use

In the old chat:

```text
handoff
```

In the new chat, paste the generated block.

Natural triggers:
- `handoff`
- `hand off`
- `new session`
- `switch session`
- `start fresh`
- `wrap up and hand off`

## Quick flow

1. work until the current chat is getting heavy
2. say `handoff`
3. copy the generated block
4. start a fresh chat
5. paste the block and continue

That is the whole workflow.

## When to use it

Use it when:
- the current chat is getting long
- you finished one slice of work and want a clean next session
- you want to avoid context rot
- you want to preserve decisions and current state without keeping the full old thread alive

## Works well with

- coding tools
- terminal agents
- GUI coding apps
- chat products like Claude chat, Cowork, or any other long-thread chat interface
- any workflow where copy-paste is easier than dragging a huge old thread forward
