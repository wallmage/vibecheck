我一直在下午就用完 Claude 额度，搞不清楚原因。后来一查才发现，我大部分 AI 编程会话有 70% 都是浪费——AI 在那儿废话连篇地预告它要干啥，明明一轮能搞定的命令非要拆成三轮，过期的上下文堆在那里每轮都被重新读一遍。

vibecheck 就是来找出这些浪费的。它读取你 24+ 种编程工具的真实会话日志，给 15 种具体的浪费模式标上美元金额，然后修掉它们。所有分析都在本地运行，不上传任何数据，没有遥测，没有服务器。

我自己的情况：月花费从 $2,816 降到了 $422。**省了 85%。**

## 安装方法

把下面这句话粘贴到你的 AI 编程工具里，按回车：

> Help me install this skill: https://github.com/wallmage/vibecheck

搞定。AI 会自动加载这个 skill，你就可以开始扫描了。

<details>
<summary>或者通过命令行手动安装</summary>

```bash
git clone https://github.com/wallmage/vibecheck.git ~/.claude/skills/vibecheck
```

然后在任意会话里输入 `/vibecheck scan`。

更新：`cd ~/.claude/skills/vibecheck && git pull`

### "Skill" 到底是什么？

就是一个纯文本文件，教你的 AI 学会一个新能力。没有二进制文件，没有后台进程，不改系统配置。vibecheck 的 skill 文件就是告诉 AI "怎么找浪费、怎么修"。删掉文件夹就彻底卸载了。

### 编程工具 vs. 聊天工具

**编程工具**（Claude Code、Codex、Cursor、OpenClaw、Copilot、Windsurf、TRAE、Qoder 等）跑在你本地，会留下会话日志。vibecheck 自动检测你装了哪些，直接扫描那些日志。

**聊天工具**（Cowork、浏览器版 Claude）跑在沙箱里，没有本地日志。vibecheck 仍然可以优化你的指令文件——大部分省钱其实就是从那儿来的嘛。你也可以粘贴一行终端命令导出 14 天的日志来做完整扫描。

### 权限说明

vibecheck 会读取你本地的会话日志，检查指令文件（`CLAUDE.md`、`AGENTS.md`、`Memory.md`）以及全局工具配置。如果你的工具有一个全局配置文件——一个文件管所有项目的那种——优化器会先处理它，因为改一处就能省所有项目的钱。改任何东西之前都会先征求你的同意。

## 隐私

所有数据都留在你的电脑上。分析过程就是一组 Python 脚本解析你的本地会话日志。没有服务器，不调用任何 API，没有数据分析。完全开源——你想的话可以逐行审查。

## 命令

| 命令 | 功能 |
|---|---|
| `/vibecheck scan` | 扫描你电脑上所有检测到的工具。输出一份统一报告，包含健康指标、排名统计、头部浪费模式和优化路线图 |
| `/vibecheck explain` | 给你讲解浪费模式，不改任何东西。纯粹学习用 |
| `/vibecheck compress` | 用 4 遍无损压缩器把你的指令文件缩小 25-50% |
| `/vibecheck monitor` | 每周跟基线做对比。在成本回弹积少成多之前就抓住它 |

扫描过程很安静：一个紧凑的进度指示器，然后输出一份干净的摘要。`Good` 表示浪费低于 10%，`Waste` 表示高于 10%。原始日志和工具的碎碎念藏在幕后，除非出了问题才会显示。

### 保持会话清爽

长对话比短对话贵——每轮新回复都要重新读前面所有内容，而且塞得太满的上下文会让 AI 犯更多错，导致更多来回折腾。

经验法则：每个会话 5-10 分钟活跃时间，30-40 轮之后上下文税就开始明显了。开新会话的时候，把持久性规则放在指令文件里（`CLAUDE.md`、`AGENTS.md`、`Memory.md`），项目背景放在小的本地文档里。新会话不等于冷启动——只是一个干净的上下文，你的知识都还在呢。

## AI 每轮对话是怎么花钱的

如果你之前没关心过 token 怎么收费，这里用大白话讲讲。

### 每轮都发生了什么

每次 AI 回复时，它都会从头重新读一遍整个对话。系统提示词、指令文件、你发的每条消息、它给的每个回复、所有工具输出——文件内容、终端结果、错误日志——全部。然后才生成新的回复。

**单轮成本 = 读入 token 数 x 输入价格 + 生成 token 数 x 输出价格**

前面几轮很便宜。第 1 轮可能只读 5,000 个 token。到第 40 轮，它要重新读 40,000+ 个 token 的累积对话——之前所有消息、所有代码片段、所有错误堆栈。那一轮的成本是第一轮的 8 倍。

问题在哪呢？**浪费是复利的。** 一轮废话不只花它自己那点钱——它赖在上下文里，后面每轮都被重读。第 10 轮多说了一句，到你收工之前还要被重读 30 遍。

### Prompt cache 有帮助，但治不了根

大多数服务商现在会缓存之前见过的内容，缓存命中的收费只有原价的 1/10。有效输入价格从 $3.00/百万 token 降到 $0.30/百万。

这当然有用。但新内容——新的工具输出、新的错误信息、AI 的每次新回复——第一次进来都是全价，然后才会被缓存。而且就算按缓存价格，浪费照样复利增长。

### 订阅制用户一样受影响

订阅用户可能觉得这跟我没关系吧？有关系的——只是痛法不一样。订阅买的是一个固定的算力池子，浪费多了池子就干得快。下午三点就被限速，不是你干多了，是你浪费多了。

Claude Pro（$20/月）大约覆盖 $200 的 API 等价算力。Claude 20x Max（$200/月）大约覆盖 $4,000。浪费越多，撞墙越快。

## 15 种浪费模式

按烧钱程度排列。光前三个就占了所有浪费的 60-70%。

### 第一梯队——三大元凶（占浪费的 60-70%）

#### 1. 空转叙述

AI 说"好的，我现在来修"或者"让我先读一下这个文件"——然后下一轮才真正干活。那轮叙述啥也没干。没有工具调用，没有代码，没有读文件。就是一句预告。

每次大约花 **$0.017**——更要命的是那 300-500 个 token 的废话会一直留在上下文里，后面每轮都被重读。在 428 个测量会话中：**每会话浪费 $1.03**，占所有浪费的 30%。每天 10 个会话的话，**光叙述就 $309/月。**

vibecheck 加了一条规则：*"每轮必须有工具调用。思考和行动放在同一轮。"* **每会话省约 $0.88。**

#### 2. 上下文腐化

会话成本是二次增长的，不是线性的。第 50 轮要重读前面 49 轮的内容。

具体对比：一个 40 轮的会话花 **$0.70**。同样的工作量拆成两个 20 轮的会话只花 **$0.60**。差出来的 $0.10 纯粹是维持一个臃肿对话的代价。实际会话平均 74 轮——**每会话浪费 $0.46**，占所有浪费的 13%。

vibecheck 教你：*"不相关的工作放到不同会话。长对话里保持紧凑。"* **每会话省约 $0.37。**

#### 3. 乒乓式调试

改了，坏了，再试，又坏了。每次失败尝试往上下文里塞约 4,000 个 token 的错误输出，那些死文本在后面每轮都被重读。三个来回：6 轮额外开销（$0.102）+ 12K token 的过时错误信息（$0.036）= **每次约 $0.14**。大约 10% 的会话会出现。**加权：$0.015/会话。**

vibecheck 加了个断路器：*"同一个文件修了 2 次还不对——停下来，重新读完整错误信息，想清楚，一次精准修复。"* **每会话省约 $0.01。**

### 第二梯队——轮次密度（占浪费的 15-20%）

明明一轮能搞定的事，非要用三轮。

#### 4. 输出太啰嗦

一个构建或测试命令往对话里倒 500 行（约 5,000 个 token）。那些 token 在会话剩余的每轮都要被重读。5K token x 12 轮剩余 x 缓存价 = **每次 $0.018**。没有缓存的话：**$0.180**——贵 10 倍。

按实际测量这其实是单项最贵的模式。构建日志、npm 输出、测试数据——几乎每个会话都有。**$1.05/会话**，占所有浪费的 31%。

修复：*"输出重定向到 /tmp/。用 --quiet 参数。最多 tail -50。"* **每会话省约 $0.89。**

#### 5. 命令不串联

一轮跑 `npm install`，下一轮跑 `npm run build`。本来 `npm install && npm run build` 一轮就行，结果重读了两次上下文。每次拆分：**$0.010**。命令密集的会话累计 **$0.14/会话**。

修复：*"安全的时候用 `&&` 串联命令。"* **每会话省约 $0.11。**

#### 6. 代码库漫游

AI 先打开 README，再打开 package.json，再翻三个配置文件和两个不相关的模块，一行代码都没写。连续五次读文件，没有编辑，没有决策。$0.085 的无用轮次 + $0.027 上下文税 = **每次 $0.112**。平均：**$0.09/会话。**

修复：先用 grep 或 glob 搜，只读相关的，一轮里批量读多个文件。**每会话省约 $0.07。**

#### 7. 编辑不批量

编辑文件 A，再编辑 B，再编辑 C——三轮。一轮并行编辑就能搞定。多出来两轮 x $0.017 = **每次 $0.034**。平均：**$0.058/会话。**

修复：*"批量处理独立的工具调用。"* **每会话省约 $0.05。**

### 第三梯队——长尾（占浪费的 5-10%）

单个看不大，但加起来不少。

#### 8. 重复读文件

同一个会话里读同一个文件两次——内容明明已经在上下文里了，AI 又去读一遍。**每次重读 $0.019**，文件平均被重读 3-4 次。**$0.066/会话。** 修复：*"已经在上下文里了。只有文件改了才重读。"* **每会话省约 $0.05。**

#### 9. Sleep/轮询循环

`sleep 5 && check_status`，反复 3-5 次。每次轮询 = 重读一遍完整上下文就为了看后台进程跑完没。4 次轮询 x $0.017 = **每次 $0.068**，**$0.043/会话。** 修复：*"用 --wait 或 run_in_background。"* **每会话省约 $0.034。**

#### 10. 无效重试

命令失败了，AI 一模一样地再跑一遍。错误输出在上下文里出现两份。**每次重试 $0.019**，**$0.080/会话。** 修复跟乒乓调试一样——*"停下来，读错误信息，换个方法试。"*

#### 11. Schema 查询

AI 去查自己的工具定义——它本来就知道的信息。白白加了 2K+ 个 token。**$0.023/会话。** "每轮必须有工具调用"那条规则顺带解决了这个问题。**每会话省约 $0.02。**

#### 12. Git 仪式

`git add` → `git status` → `git commit` → `git push`。四轮。`git add -A && git commit -m "msg" && git push` 一轮搞定。**每次 $0.044**，但比你想的少见——**$0.003/会话。** 修复：用 `&&` 串联。

### 第四梯队——常驻 Agent

成本模型不同。像 OpenClaw 这样的 Agent 会定期唤醒，浪费按天而不是按会话计算。

#### 13. 空转心跳

Agent 每 5 分钟醒一次，重读整个工作区，发现没事干，接着睡。每天 288 次唤醒，约 97% 是空转。280 次空转唤醒 x 每次 $0.04 = **$11.20/天（$336/月）**，啥也没干。

修复：*"最低 30 分钟心跳间隔。没有待处理触发器就跳过。"* 降到每天约 48 次唤醒。**每天省 $8-10（$240-300/月）。**

#### 14. 工作区文件膨胀

35,000 个 token 的人设文件（SOUL.md、AGENTS.md 等）每次唤醒都要重读。教程、辅导、理念——全是写给人看的，不是给跑任务的 AI 看的。**$5.76/天（$173/月）** 光花在配置文件上。

vibecheck 压缩它们：35K → 12-15K token。行为规则一个不少，去掉给人看的填充内容。**每天省 $3-4（$90-120/月）。**

#### 15. 记忆堆积

会话历史无限增长。100+ 条记忆条目每次唤醒都重读，包括几周前的、早就没用的东西。**$3.17/天（$95/月）** 花在过期记忆上。

修复：*"超过 50 条就归档，做总结，重新开始。"* **每天省 $2-3（$60-90/月）。**

## 优化工具箱

光说哪儿浪费不够，得修啊。

### 指令文件压缩

你的指令文件（`CLAUDE.md`、`AGENTS.md`、`Memory.md`，不管你的工具叫它什么名字）每轮都会被读一遍。这是一笔固定税，你做什么都要交。指令文件太大就像每条路上都有收费站，走哪都要交钱。

vibecheck 有一个 4 遍无损压缩器——23 种技术，"无损"就是字面意思：不删任何事实。同样的信息，更少的 token。

| 遍次 | 做什么 | 省多少 |
|---|---|---|
| **第 1 遍——机械压缩** | 去掉 markdown 格式，表格转紧凑形式，合并列表项 | 10-15% |
| **第 2 遍——保真压缩** | 去重重复事实，压缩代码示例，合并冗长描述 | 15-25% |
| **第 3 遍——高保真** | 去掉人类需要但 AI 不需要的教程和辅导文本 | 10-15% |
| **第 4 遍——电报模式** | 全面简写重写，只保留 AI 能读懂的形式。密度很高，但需要你明确授权才会执行 | 15-25% |

一个 10,000 token 的指令文件压缩到 6,000，每个会话省 $0.044。每天 10 个会话的话：**$0.44/天（$13/月）**，光压缩就省这么多。

### 输出抑制

输出 token 的价格是输入的 5 倍（Sonnet 4.6 上 $15 vs. $3/百万）。AI 没事就打印完整代码块或你没要的 diff？很贵的。vibecheck 加一条：*"不问不贴代码和 diff。"* **每会话省约 $0.047。**

### 成本监控

`/vibecheck monitor` 会给你的会话状况拍快照，以后每次跑都跟基线对比。新的指令文件引入了啰嗦的写法？换了项目习惯变了？监控器在成本回弹积少成多之前就能发现。

<details>
<summary><strong>方法论</strong></summary>

所有成本估算使用上述参考场景。关键假设：

- **95% prompt cache 命中率** —— 快速编程会话的典型值。如果你打字慢、轮次间隔长，命中率会更低，成本更高。
- **25 轮有效轮次/会话** —— 浪费严重的会话会因为叙述、重试和命令不串联多出 8-12 轮。
- **每轮增长 600 token** —— 输出啰嗦的会话可能达到 1,000-2,000 token/轮。
- **有效输入费率：$0.435/1M** —— 95% 缓存（$0.30/1M）+ 5% 未缓存（$3.00/1M）的混合费率。
- **上下文税费率：$0.30/1M** —— 永久上下文新增内容按缓存输入价计算。

以上都是保守估算。实际省的钱往往更多——会话越长、指令文件越大、调试越频繁，省得越多。
</details>
