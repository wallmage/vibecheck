### 什么是 skill？

相当于给 AI 的一张配方卡。它不会修改任何东西，也不会安装任何程序——只是一个文本文件，告诉 AI "该如何发现浪费并修复它"。随时可以删除。

### 编程工具 vs 聊天工具

**编程工具**（Claude Code、Codex、Cursor、OpenClaw、Copilot、Windsurf、TRAE、Qoder 等）运行在你的机器上——vibecheck 会自动识别你的工具并扫描会话日志。

**聊天工具**（Cowork、浏览器端等）运行在沙箱中。vibecheck 仍可优化你的指令文件（这是最主要的收益），或者你粘贴一条终端命令，将 14 天的日志复制过来进行完整扫描。

### 权限说明

vibecheck 会读取并编辑你的指令文件（CLAUDE.md、.cursorrules 等），每次修改前都会征求你的确认。

## 隐私

你的数据不会离开本机。没有服务器、没有 API、没有遥测。完全开源。

## 轮次如何产生费用

每一轮对话，AI 都会重新读取整个对话内容——系统提示、指令文件、所有历史消息、所有工具输出——然后生成回复。

**轮次费用 = 输入 token 数 × 输入单价 + 输出 token 数 × 输出单价**

早期轮次较便宜（context 较小）。后期轮次很贵（之前所有内容都要重新读取）。这就是为什么浪费会复利叠加——一轮浪费会让后续每一轮都更贵，因为浪费的内容一直留在 context 中。

Prompt caching 可将已读取内容的输入成本降低 10 倍。大多数工具会自动启用。

**订阅用户：** 你不会直接看到 API 价格，但浪费会更快消耗你的消息配额。Claude Pro（$20/月）约等价于 $200 的 API 用量。Max（$200/月）约等价于 $4,000。

## 15 种浪费模式

### 第一层级 —— 三大主因（占浪费的 60-70%）

#### 1. 空转叙述（Idle Narration）

**是什么。** AI 先说"好的，我来修复"或"让我先读取文件"——然后才在下一轮真正开始工作。叙述那一轮什么都没做：没有工具调用，没有代码，没有文件读取。

**损耗。** 每轮叙述花费 **$0.031**（重新读取 context + ~500 tokens 的状态文字）。大多数会话有 5 条这样的叙述：**每会话浪费 $0.165**——占账单的 17%，却什么都没产出。每天 10 个会话：**每天 $1.65（每月 $50）** 全耗在叙述上。

**修复方式。** vibecheck 添加规则：*"每轮必须有工具调用。禁止纯叙述。思考和行动在同一轮完成。"* 彻底消除空转叙述。**每会话节省 $0.15-0.18。**

#### 2. Context 腐烂（Context Rot）

**是什么。** 对话越长，费用增长越快。第 50 轮需要重新读取前 49 轮的全部内容。会话总费用随长度呈二次方增长。

**损耗。** 一次 40 轮会话：**$1.89**。两次 20 轮会话（同等工作量）：**$1.22**。差额 **$0.67** 没有任何价值。100 轮时：单次会话 **$5.62**，四次 25 轮会话 **$3.84**。不拆分白白浪费 **$1.78**。

**修复方式。** 培养习惯：*"不相关任务之间使用 /clear 或 /compact。开启新对话。"* **对有长会话习惯的用户，每会话节省 $0.30-0.70。**

#### 3. 乒乓调试（Ping-Pong Debugging）

**是什么。** 修复、报错、重试、再报错。每次失败都向 context 追加错误输出（每轮约 4K tokens），后续每一轮都要重新读取。

**损耗。** 三次失败循环：6 轮额外对话（$0.252）+ 12K tokens 的无效错误信息（context 税 $0.036）。**每次约 $0.29**。约 1/3 的会话会出现此问题。**加权后：~$0.10/会话。**

**修复方式。** 添加规则：*"同一文件失败两次后：停下来，完整读取错误，思考，再进行一次精准修复。"* **每次节省约 $0.20。**

### 第二层级 —— 轮次密度（占浪费的 15-20%）

#### 4. 冗长工具输出（Verbose Tool Output）

**是什么。** 构建/测试命令将 500 行（约 5K tokens）输出到对话中。这些 token 在后续每一轮都会被重新读取。

**损耗。** 5K tokens × 剩余 12 轮 × $0.30/1M = 每次 context 税 **$0.018**。每会话发生 2-3 次。不使用 caching 时：**$0.180/次** —— 高 10 倍。**总计：每会话 $0.04-0.05。**

**修复方式。** 添加规则：*"将构建/测试输出重定向到 /tmp/，使用 --quiet 标志，最多 tail -50。"* **每会话节省 $0.03-0.05。**

#### 5. 未链式命令（Unchained Commands）

**是什么。** 一轮运行 `npm install`，下一轮运行 `npm run build`。用 `&&` 链式执行只需一轮，现在却读取了两次 context。

**损耗。** 每次拆分：**$0.023**。典型会话有 3-4 次拆分。**总计：每会话 $0.07-0.09。**

**修复方式。** 添加规则：*"安全时用 `&&` 链接命令。"* **每会话节省 $0.06-0.08。**

#### 6. 漫游代码库（Codebase Wandering）

**是什么。** AI 在真正动手之前，一个接一个地打开文件——README、package.json、各种配置——连续读取 5 个以上文件才开始第一次编辑。

**损耗。** 5 次不必要的读取：$0.190 轮次费用 + $0.027 context 税 = **每次 $0.217**。约 25% 的会话会出现此问题。**加权后：~$0.054/会话。**

**修复方式。** 鼓励有针对性的搜索（先用 grep/glob），并在一轮中批量读取多个文件。**每次节省约 $0.15。**

#### 7. 未批量编辑（Unbatched Edits）

**是什么。** 先编辑文件 A，再编辑文件 B，再编辑文件 C——三轮，而一轮并行编辑就能搞定。

**损耗。** 2 轮额外对话 × $0.038 = **每次 $0.076**。约 60% 的会话会出现。**加权后：~$0.046/会话。**

**修复方式。** 添加规则：*"批量处理独立工具调用（每轮多次 Read/Edit）。"* **每会话节省约 $0.04。**

### 第三层级 —— 长尾损耗（占浪费的 5-10%）

#### 8. 重复读取文件（File Re-reads）

**是什么。** 同一个文件在一次会话中被读取两次。第一次读取后内容已在 context 中。

**损耗。** 1 轮浪费 + 重复内容 = **每次 $0.043**。每会话 1-2 次。**加权后：~$0.039/会话。**

**修复方式。** 添加规则：*"第一次读取后内容已在 context 中。仅在文件变更后重新读取。"* **每会话节省约 $0.03。**

#### 9. 轮询循环（Sleep/Poll Loops）

**是什么。** `sleep 5 && check_status`，重复 3-5 次。每次轮询都重新读取完整 context。

**损耗。** 4 次轮询 × $0.038 = **每次 $0.152**。约 20% 的会话会出现。**加权后：~$0.030/会话。**

**修复方式。** 添加规则：*"使用 --wait 标志或 run_in_background。"* **每次节省约 $0.12。**

#### 10. 失败重试（Failed Retries）

**是什么。** 命令失败，AI 原样再运行一次相同命令。错误输出现在在 context 中出现了两次。

**损耗。** **每次 $0.042**。约 30% 的会话会出现。**加权后：~$0.013/会话。**

**修复方式。** 与乒乓调试相同的规则：*"停下来，完整读取错误，思考，再进行一次精准修复。"*

#### 11. Schema 查询（Schema Lookups）

**是什么。** AI 去查找自己的工具定义——而这是它本来就知道的信息。向 context 追加 2K+ tokens。

**损耗。** **每次 $0.052**。约 40% 的会话会出现。**加权后：~$0.021/会话。**

**修复方式。** "每轮必须有工具调用"规则会遏制探索性轮次。**每会话节省约 $0.02。**

#### 12. Git 繁文缛节（Git Ceremony）

**是什么。** `git add` → `git status` → `git commit` → `git push`，四轮。而 `git add -A && git commit -m "msg" && git push` 只需一轮。

**损耗。** 3 轮额外对话 + git 输出 = **每次 $0.098**。约 70% 的会话会出现。**加权后：~$0.069/会话。**

**修复方式。** 添加规则：*"用 `&&` 链接 git 命令。"* **每会话节省约 $0.06。**

### 第四层级 —— 常驻 Agent（OpenClaw 等）

不同的费用模型：每次唤醒费用 × 每日唤醒次数。

#### 13. 空闲心跳（Idle Heartbeats）

**是什么。** Agent 每 5 分钟唤醒一次，重新读取完整工作区，什么都没发现。每天 288 次唤醒，约 97% 是空转。

**损耗。** 每天 280 次空转 × $0.04 = **每天 $11.20（每月 $336）**，什么都没做。

**修复方式。** *"最短心跳间隔 30 分钟。无触发时跳过。"* 每天减少到约 48 次唤醒。**每天节省 $8-10（每月 $240-300）。**

#### 14. 工作区文件膨胀（Workspace File Bloat）

**是什么。** 每次唤醒重新读取 35K tokens 的配置文件（SOUL.md、AGENTS.md 等）。AI 只需要其中的行为规则——教程和说明是给人看的。

**损耗。** **每天 $5.76（每月 $173）** 只是在读配置文件。

**修复方式。** 压缩工作区文件：35K → 12-15K tokens。**每天节省 $3-4（每月 $90-120）。**

#### 15. 记忆积累（Memory Accumulation）

**是什么。** 会话历史不经修剪持续增长。每次唤醒重新读取 100+ 条记录。

**损耗。** **每天 $3.17（每月 $95）** 在读取过期记忆。

**修复方式。** *"超过 50 轮后归档，总结，重新开始。"* **每天节省 $2-3（每月 $60-90）。**

## 附加：优化工具

### 指令文件压缩

指令文件每轮都要读取——无论任务内容如何，这是一笔固定税。vibecheck 内置一个 4 轮无损压缩器（23 种技术），可将文件体积压缩 25-50%：

- **第 1 轮（机械处理）：** 去除 markdown 装饰，转换表格，合并列表项。约 10-15%。
- **第 2 轮（保留事实）：** 去重，压缩代码示例。约 15-25%。
- **第 3 轮（高保真）：** 移除人类需要但 AI 不需要的教程和说明文字。约 10-15%。
- **第 4 轮（电报体）：** 针对纯 AI 文件的全面简写改写。约 15-25%（需明确授权）。

一个 10K token 的文件压缩到 6K，每会话节省 $0.057。每天 10 个会话：**每天 $0.57（每月 $17）。**

### 输出抑制

输出 token 的价格是输入的 5 倍（Sonnet 上 $15 vs $3/MTok）。AI 主动展示你没有要求的完整代码块和 diff，每会话浪费约 **$0.047**。vibecheck 添加规则：*"除非被要求，否则不输出代码/diff。"*

### 费用监控

`/vibecheck monitor` 会为你的会话状况创建快照，并在后续运行时与基准对比，在损耗变严重之前及时发现回退。

所有费用估算均基于上述参考场景。主要假设：

- **90% prompt cache 命中率** —— 适用于快节奏的编程会话。节奏较慢的会话实际费用会更高。
- **每会话 25 个有效轮次** —— 低效会话因叙述、重试、未链式命令会额外增加 8-12 轮。
- **每轮增长 3,000 tokens** —— 冗长会话可达 4,000-5,000。
- **有效输入单价：$0.57/1M** —— 90% 已缓存（$0.30）+ 10% 未缓存（$3.00）的混合价格。
- **Context 税率：$0.30/1M** —— 针对永久性 context 增量的已缓存输入单价。

估算结果偏保守。对于长会话、大型指令文件或频繁调试的用户，实际节省可能超过预期。
