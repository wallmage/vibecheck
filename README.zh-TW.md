### 什麼是 skill？

給你的 AI 用的食譜卡。它不修改任何東西，也不安裝任何東西。只是一個文字檔，告訴 AI「怎麼找出浪費並修掉它」。隨時刪掉都行。

### 開發工具 vs 聊天工具

**開發工具**（Claude Code、Codex、Cursor、OpenClaw、Copilot、Windsurf、TRAE、Qoder 等）跑在你的機器上——vibecheck 會自動偵測你的工具並掃描你的 session 記錄。

**聊天工具**（Cowork、瀏覽器版）跑在沙盒裡。vibecheck 仍可優化你的指令檔（這是主要效益所在），或者貼上一行終端機指令，把最近 14 天的記錄複製過來做完整掃描。

### 權限

vibecheck 會讀取並編輯你的指令檔（CLAUDE.md、.cursorrules 等）。每次變更前都會先問你。

## 隱私

你的資料不離開你的機器。沒有伺服器、沒有 API、沒有遙測。開源。

## 輪次如何轉化為費用

每一輪，你的 AI 都會重新讀取整段對話——系統提示、指令檔、所有先前訊息、所有工具輸出——然後生成回應。

**每輪費用 = 輸入 token 數 × 輸入費率 + 輸出 token 數 × 輸出費率**

早期輪次便宜（context 小）。後期輪次昂貴（之前所有內容都得重讀）。這就是為什麼浪費會複利滾大——浪費的輪次會讓所有後續輪次更貴，因為那些廢內容還留在 context 裡。

Prompt caching 能讓先前出現過的內容輸入費用降至 1/10。大多數工具會自動使用。

**訂閱用戶：** 你不會直接看到 API 價格，但浪費會更快燒光你的訊息配額。Claude Pro（$20/月）相當於約 $200 的 API 價值；Max（$200/月）相當於約 $4,000。

## 15 種浪費模式

### Tier 1 — 三大元凶（佔浪費的 60-70%）

#### 1. 空轉旁白（Idle Narration）

**是什麼。** AI 說「好，我來修」或「讓我先讀一下這個檔案」——然後下一輪才真正動手。那個旁白輪次什麼都沒做：沒有工具呼叫、沒有程式碼、沒有讀檔。

**浪費多少。** 每個旁白輪次花費 **$0.031**（context 重讀 + 約 500 tokens 的狀態文字）。大多數 session 有 5 次這樣的情況：**每 session 浪費 $0.165**——你帳單的 17% 毫無產出。一天 10 個 session：**光旁白就花 $1.65/天（$50/月）。**

**修法。** vibecheck 加入：*「每一輪都必須有工具呼叫。不說旁白。思考與行動在同一輪完成。」* 徹底消除旁白。**省下 $0.15-0.18/session。**

#### 2. Context 腐爛（Context Rot）

**是什麼。** 對話越長，費用遞增越快。第 50 輪要重讀前 49 輪。Session 總費用隨長度呈二次方增長。

**浪費多少。** 一個 40 輪 session：**$1.89**。兩個 20 輪 session（相同的工作量）：**$1.22**。差價——**$0.67**——什麼都沒換到。100 輪時：一個 session 花 **$5.62**，而四個 25 輪 session 只需 **$3.84**。不拆開就白燒 **$1.78**。

**修法。** 教學：*「不相關的任務之間用 /clear 或 /compact。重開新對話。」* **對習慣開長 session 的用戶，省下 $0.30-0.70/session。**

#### 3. 乒乓除錯（Ping-Pong Debugging）

**是什麼。** 修了又壞、再修再壞。每次失敗都把錯誤輸出加進 context（約每個循環 4K tokens），之後每一輪都得重讀。

**浪費多少。** 三次失敗循環：6 個額外輪次（$0.252）＋12K tokens 的廢錯誤資訊（$0.036 context 稅）。**合計：每次發作約 $0.29。** 約 1/3 的 session 會發生。**加權後：約 $0.10/session。**

**修法。** 加入：*「同一個檔案修了 2 次還沒好：停下來，完整重讀錯誤，思考，一次到位地修。」* **每次發作省下約 $0.20。**

### Tier 2 — 輪次密度（佔浪費的 15-20%）

#### 4. 冗長工具輸出（Verbose Tool Output）

**是什麼。** 建置/測試指令把 500 行（約 5K tokens）倒進對話裡。那些 token 在每一輪後續都得重讀。

**浪費多少。** 5K tokens × 12 個剩餘輪次 × $0.30/1M = **每次 $0.018** context 稅。每個 session 發生 2-3 次。沒有 caching 的情況：**每次 $0.180**——貴 10 倍。**合計：$0.04-0.05/session。**

**修法。** 加入：*「把建置/測試輸出導向 /tmp/，使用 --quiet 旗標，最多 tail -50。」* **省下 $0.03-0.05/session。**

#### 5. 未串接指令（Unchained Commands）

**是什麼。** 一輪 `npm install`，下一輪 `npm run build`。兩次 context 重讀，明明 `&&` 串在一起就只需一次。

**浪費多少。** 每次拆開：**$0.023**。典型 session 有 3-4 次拆開。**合計：$0.07-0.09/session。**

**修法。** 加入：*「安全的情況下用 `&&` 串接指令。」* **省下 $0.06-0.08/session。**

#### 6. 漫遊程式庫（Codebase Wandering）

**是什麼。** AI 一個接一個開檔案——README、package.json、設定檔——什麼都不做就先這樣。連續五個以上的讀取，才做第一次編輯。

**浪費多少。** 五次不必要的讀取：$0.190 輪次費＋$0.027 context 稅 = **每次發作 $0.217**。約 25% 的 session 會發生。**加權後：約 $0.054/session。**

**修法。** 鼓勵針對性搜尋（先用 grep/glob），並且每輪批次多個讀取。**每次發作省下約 $0.15。**

#### 7. 逐一編輯（Unbatched Edits）

**是什麼。** 編輯檔案 A，再編輯 B，再編輯 C——三輪，明明一輪平行編輯就能完成。

**浪費多少。** 2 個額外輪次 × $0.038 = **每次 $0.076**。約 60% 的 session 會發生。**加權後：約 $0.046/session。**

**修法。** 加入：*「批次處理獨立的工具呼叫（每輪多個 Read/Edit）。」* **省下約 $0.04/session。**

### Tier 3 — 長尾（佔浪費的 5-10%）

#### 8. 重複讀檔（File Re-reads）

**是什麼。** 同一個 session 中同一個檔案被讀了兩次。第一次讀完，內容就已經在 context 裡了。

**浪費多少。** 1 個浪費輪次＋重複內容 = **每次重讀 $0.043**。每個 session 1-2 次。**加權後：約 $0.039/session。**

**修法。** 加入：*「第一次讀完後內容就在 context 裡了。只有檔案變動時才重讀。」* **省下約 $0.03/session。**

#### 9. 輪詢迴圈（Sleep/Poll Loops）

**是什麼。** `sleep 5 && check_status`，重複 3-5 次。每次輪詢都重讀完整的 context。

**浪費多少。** 4 次輪詢 × $0.038 = **每次發作 $0.152**。約 20% 的 session 會發生。**加權後：約 $0.030/session。**

**修法。** 加入：*「使用 --wait 旗標或 run_in_background。」* **每次發作省下約 $0.12。**

#### 10. 無效重試（Failed Retries）

**是什麼。** 指令失敗，AI 一模一樣地再跑一次。錯誤輸出現在在 context 裡出現兩次。

**浪費多少。** **每次重試 $0.042**。約 30% 的 session 會發生。**加權後：約 $0.013/session。**

**修法。** 與乒乓除錯相同的規則：*「停下來，重讀錯誤，思考，一次到位地修。」*

#### 11. Schema 查找（Schema Lookups）

**是什麼。** AI 查找自己的工具定義——那是它本來就有的資訊。把 2K+ tokens 加進 context。

**浪費多少。** **每次查找 $0.052**。約 40% 的 session 會發生。**加權後：約 $0.021/session。**

**修法。** 「每一輪都必須有工具呼叫」這條規則抑制了探索性輪次。**省下約 $0.02/session。**

#### 12. Git 儀式（Git Ceremony）

**是什麼。** `git add` → `git status` → `git commit` → `git push`，四輪。`git add -A && git commit -m "msg" && git push` 只需一輪。

**浪費多少。** 3 個額外輪次＋git 輸出 = **每次 $0.098**。約 70% 的 session 會發生。**加權後：約 $0.069/session。**

**修法。** 加入：*「用 `&&` 串接 git 指令。」* **省下約 $0.06/session。**

### Tier 4 — 常駐 Agent（OpenClaw 等）

不同的費用模式：每次喚醒費用 × 每日喚醒次數。

#### 13. 閒置心跳（Idle Heartbeats）

**是什麼。** Agent 每 5 分鐘喚醒一次，重讀整個工作區，什麼都沒發現就結束。每天 288 次喚醒，約 97% 是閒置的。

**浪費多少。** 每天 280 次閒置喚醒 × $0.04 = **$11.20/天（$336/月）**，什麼事都沒做。

**修法。** *「最低心跳間隔 30 分鐘。沒有觸發條件時跳過。」* 降至每天約 48 次喚醒。**省下 $8-10/天（$240-300/月）。**

#### 14. 工作區檔案膨脹（Workspace File Bloat）

**是什麼。** 35K tokens 的人格設定檔（SOUL.md、AGENTS.md）每次喚醒都重讀。AI 只需要行為規則——教學和引導文字是給人看的。

**浪費多少。** **$5.76/天（$173/月）**，只是在讀設定檔。

**修法。** 壓縮工作區檔案：35K → 12-15K tokens。**省下 $3-4/天（$90-120/月）。**

#### 15. 記憶體累積（Memory Accumulation）

**是什麼。** Session 歷史不斷增長，從不修剪。每次喚醒都重讀 100+ 條記錄。

**浪費多少。** **$3.17/天（$95/月）**，在讀陳舊的記憶。

**修法。** *「超過 50 輪後封存，做摘要，重新開始。」* **省下 $2-3/天（$60-90/月）。**

## 附加功能：優化工具

### 指令檔壓縮

你的指令檔每一輪都會被讀取——這是你無論做什麼任務都得支付的固定稅。vibecheck 內含一個 4 道步驟的無損壓縮器（23 種技術），能將檔案大小縮小 25-50%：

- **第 1 道（機械式）：** 去除 markdown 裝飾、轉換表格、合併條列項目。約 10-15%。
- **第 2 道（保留事實）：** 去重複事實、壓縮程式碼範例。約 15-25%。
- **第 3 道（高保真）：** 移除人類需要、AI 不需要的教學和引導文字。約 10-15%。
- **第 4 道（電報式）：** 純 AI 閱讀的檔案進行全面簡寫改寫。約 15-25%（需取得許可才執行）。

一個 10K token 的檔案壓縮到 6K，每個 session 省下 $0.057。一天 10 個 session：**$0.57/天（$17/月）。**

### 輸出抑制

輸出 token 的費用是輸入的 5 倍（Sonnet 上是 $15 vs $3/MTok）。AI 在你沒要求的情況下顯示完整程式碼區塊和差異比對，每個 session 浪費約 **$0.047**。vibecheck 加入：*「未經要求不顯示程式碼/差異比對。」*

### 費用監控

`/vibecheck monitor` 會為你的 session 特徵拍個快照，並在後續執行時與基準值比較。在燒錢之前就抓到退步。

<summary>研究方法</summary>

所有費用估算均採用上述參考情境。主要假設：

- **90% prompt cache 命中率** — 快節奏編碼 session 的典型值。節奏較慢的 session 費用會更高。
- **每 session 25 個有效輪次** — 有浪費的 session 會因為旁白、重試、未串接指令等增加 8-12 個額外輪次。
- **每輪增長 3,000 tokens** — 冗長的 session 可達 4,000-5,000。
- **有效輸入費率：$0.57/1M** — 混合計算：90% 有 cache（$0.30）+ 10% 無 cache（$3.00）。
- **Context 稅率：$0.30/1M** — 永久加入 context 的內容適用的 cached 輸入費率。

估算偏保守。對於開長 session、有大型指令檔、或重度除錯的使用者，實際節省可能超過預測值。
